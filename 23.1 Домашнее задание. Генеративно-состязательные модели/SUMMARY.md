Конспект по теме домашнего задания «23.1 Генеративно-состязательные модели».

## 1. Ключевые концепции

### Что такое GAN

**GAN (Generative Adversarial Network)** — генеративная модель, состоящая из двух
нейросетей, которые обучаются *одновременно* и *соревнуясь*. Идея предложена Яном
Гудфеллоу в 2014 году ([Goodfellow et al., 2014](https://arxiv.org/abs/1406.2661)).
GAN не оценивает плотность вероятности напрямую, а учится *семплировать* объекты
из распределения данных: после обучения генератор умеет по случайному шуму выдавать
правдоподобные изображения.

### Генератор (Generator, G)

**Генератор** превращает случайный вектор шума `z` в объект (в нашем случае —
изображение лица). Его задача — научиться выдавать настолько реалистичные картинки,
чтобы дискриминатор не смог отличить их от настоящих. Генератор никогда не видит
реальные данные напрямую: единственный сигнал, по которому он учится, — это
обратная связь от дискриминатора.

### Дискриминатор (Discriminator, D)

**Дискриминатор** — бинарный классификатор. Он получает на вход изображение и
выдаёт вероятность того, что это изображение настоящее (метка 1), а не
сгенерированное (метка 0). По сути это «критик», который заставляет генератор
улучшаться: чем точнее дискриминатор ловит фейки, тем сильнее давление на генератор.

### Состязательная игра (minimax)

Обучение GAN — это игра двух игроков с нулевой суммой. Генератор и дискриминатор
оптимизируют один и тот же функционал, но в *противоположных направлениях*:

```
min_G  max_D   E_x~p_data [log D(x)]  +  E_z~p_z [log (1 - D(G(z)))]
```

Дискриминатор `D` хочет **максимизировать** этот функционал (правильно отличать
реальное от фейка), генератор `G` хочет его **минимизировать** (заставить `D`
ошибаться). В теоретическом оптимуме генератор воспроизводит истинное
распределение данных, а дискриминатор не может отличить реальное от фейка и для
любого входа выдаёт 0.5.

### Роль шумового латентного вектора

Вход генератора — случайный вектор `z` (латентный вектор, latent vector),
обычно из стандартного нормального распределения `N(0, 1)`. Этот шум — источник
*разнообразия*: разные `z` дают разные изображения. В DCGAN-генераторе для лиц
вектор имеет форму `latent_size x 1 x 1` (в решении `latent_size = 128`), то есть
это «изображение» 1x1 с большим числом каналов, которое транспонированные свёртки
постепенно «раскручивают» до полноразмерной картинки 3x128x128.

### Бинарная кросс-энтропия как лосс

Поскольку дискриминатор решает задачу бинарной классификации, естественный лосс —
**бинарная кросс-энтропия** (Binary Cross-Entropy, BCE):

```
BCE(p, y) = -[ y * log(p) + (1 - y) * log(1 - p) ]
```

где `p` — предсказанная вероятность «реальности», `y` — целевая метка (1 или 0).
В PyTorch это `nn.BCELoss()` (применяется к выходу после `Sigmoid`). И дискриминатор,
и генератор используют BCE — отличаются только целевые метки `y`, которые им
подставляются (см. раздел 4).

### Почему лоссы GAN не сходятся к нулю

В обычной supervised-задаче падение лосса к нулю означает успех. В GAN это **не
так**: генератор и дискриминатор — антагонисты, и улучшение одного автоматически
ухудшает лосс другого. Если бы лосс дискриминатора ушёл в ноль, генератор перестал
бы получать полезный градиент. Здоровое обучение GAN выглядит как **колебание
обоих лоссов в разумном диапазоне** вокруг некоторого равновесия, а не монотонное
падение. Именно так и ведёт себя обучение в этом задании (раздел 6).

### Проблемы обучения GAN

- **Mode collapse (коллапс мод).** Генератор находит несколько «удобных» картинок,
  которые обманывают дискриминатор, и выдаёт почти только их, теряя разнообразие.
  Распределение `p_data` имеет много мод (разные лица), а генератор покрывает
  лишь часть.
- **Нестабильность обучения.** Равновесие двух сетей хрупкое: если дискриминатор
  становится слишком сильным, лосс генератора «взрывается»; если слишком слабым —
  генератору нечему учиться. Лоссы могут осциллировать или расходиться.
- **Vanishing gradients (затухающие градиенты).** Когда дискриминатор почти
  идеален, `D(G(z)) ~ 0`, и градиент исходного лосса `log(1 - D(G(z)))` для
  генератора почти нулевой — генератор перестаёт обучаться. На практике вместо
  минимизации `log(1 - D(G(z)))` генератор **максимизирует** `log(D(G(z)))`
  (non-saturating loss), что эквивалентно подстановке генератору целевой метки 1
  для фейков — именно так сделано в решении.

### DCGAN — устойчивая свёрточная архитектура

**DCGAN (Deep Convolutional GAN)**, [Radford et al., 2015](https://arxiv.org/abs/1511.06434),
— набор архитектурных рецептов, которые делают обучение GAN на изображениях
заметно стабильнее:

- **`ConvTranspose2d`** в генераторе для апсемплинга (вместо pooling/upsample),
  **`Conv2d` со stride** в дискриминаторе для даунсемплинга — сеть сама учится,
  как менять разрешение.
- **`BatchNorm`** почти во всех слоях обеих сетей — стабилизирует обучение,
  выравнивает распределения активаций. Не ставится на выход генератора и на вход
  дискриминатора.
- Активации: **`ReLU`** в генераторе (кроме выхода), **`LeakyReLU(0.2)`** в
  дискриминаторе (отрицательный наклон не даёт градиенту полностью занулиться).
- **`Tanh`** на выходе генератора — изображения в диапазоне `[-1, 1]`.
- **Инициализация весов `N(0, 0.02)`** для свёрток, `N(1, 0.02)` для весов
  BatchNorm, нули для bias — эмпирически найденный режим, в котором GAN заводится
  стабильно.

### Label smoothing (сглаживание меток)

Простой стабилизирующий приём: при обучении дискриминатора реальным изображениям
ставят целевую метку не ровно `1.0`, а слегка меньше — например `0.9`. Это не
даёт дискриминатору становиться «слишком уверенным», ограничивает резкие
градиенты и снижает риск vanishing gradients у генератора. Фейкам метку оставляют
`0.0`. В решении используется `real_label_value = 0.9`.

## 2. Данные и предобработка

**Датасет.** Подмножество [Flickr Faces (FFHQ)](https://github.com/NVlabs/ffhq-dataset)
— фотографии лиц людей. Оригинал в разрешении 1024x1024 слишком велик для быстрого
обучения, поэтому берётся уменьшенная выборка. В прогоне решения загружено
**3143 изображения**.

**Предобработка** (функция `get_dataloader`, трансформации через `tt.Compose`):

| Шаг | Назначение |
|---|---|
| `tt.Resize(128)` | уменьшение до рабочего разрешения 128x128 (1024 слишком велик) |
| `tt.CenterCrop(128)` | центральный кроп — гарантирует квадратную картинку |
| `tt.ToTensor()` | перевод в тензор, значения пикселей в `[0, 1]` |
| `tt.Normalize([0.5]x3, [0.5]x3)` | нормировка в `[-1, 1]` |

Нормировка в `[-1, 1]` **обязательно согласована** с активацией `Tanh` на выходе
генератора: генератор и реальные изображения должны жить в одном диапазоне, иначе
дискриминатор различал бы их тривиально — по диапазону значений. Картинки
загружаются через `ImageFolder`, `DataLoader` строится с `shuffle=True` и
`drop_last=True` (отбрасывание последнего неполного батча — BatchNorm стабильнее
на батчах фиксированного размера). При `image_size=128`, `batch_size=64` получается
**49 батчей за эпоху** формы `(64, 3, 128, 128)`, диапазон значений `[-1.0, 1.0]`.

## 3. Архитектура генератора и дискриминатора

Обе сети — `nn.Sequential`, выстроены по рецептам DCGAN.

### Генератор: `latent 128x1x1 -> изображение 3x128x128`

Принцип: «раскрутить» вектор шума в изображение, поэтапно увеличивая
пространственный размер и уменьшая число каналов.

| Блок | Слой | Размер на выходе |
|---|---|---|
| вход | латентный вектор | `[B, 128, 1, 1]` |
| 1 | `ConvTranspose2d(128->512, 4, stride=1, pad=0)` + BN + ReLU | `[B, 512, 4, 4]` |
| 2 | `ConvTranspose2d(512->256, 4, stride=2, pad=1)` + BN + ReLU | `[B, 256, 8, 8]` |
| 3 | `ConvTranspose2d(256->128, 4, stride=2, pad=1)` + BN + ReLU | `[B, 128, 16, 16]` |
| 4 | `ConvTranspose2d(128->64,  4, stride=2, pad=1)` + BN + ReLU | `[B, 64, 32, 32]` |
| 5 | `ConvTranspose2d(64->64,   4, stride=2, pad=1)` + BN + ReLU | `[B, 64, 64, 64]` |
| 6 | `ConvTranspose2d(64->3,    4, stride=2, pad=1)` + **Tanh** | `[B, 3, 128, 128]` |

- **`ConvTranspose2d`** (транспонированная свёртка) — обучаемый апсемплинг: каждый
  блок с `stride=2` удваивает высоту и ширину. Первый блок с `stride=1, padding=0`
  и ядром 4x4 переводит 1x1 -> 4x4.
- **BatchNorm + ReLU** после каждого блока — стабилизация и нелинейность; на
  выходном слое их нет.
- **`Tanh`** на выходе зажимает значения в `[-1, 1]` — тот же диапазон, что у
  нормированных реальных изображений.

### Дискриминатор: `изображение 3x128x128 -> вероятность [0,1]`

Принцип, зеркальный генератору: поэтапно уменьшать пространственный размер и
наращивать число каналов, пока не останется один скаляр-вероятность.

| Блок | Слой | Размер на выходе |
|---|---|---|
| вход | изображение | `[B, 3, 128, 128]` |
| 1 | `Conv2d(3->64, 4, stride=2, pad=1)` + LeakyReLU(0.2) | `[B, 64, 64, 64]` |
| 2 | `Conv2d(64->128, 4, stride=2, pad=1)` + BN + LeakyReLU | `[B, 128, 32, 32]` |
| 3 | `Conv2d(128->256, 4, stride=2, pad=1)` + BN + LeakyReLU | `[B, 256, 16, 16]` |
| 4 | `Conv2d(256->512, 4, stride=2, pad=1)` + BN + LeakyReLU | `[B, 512, 8, 8]` |
| 5 | `Conv2d(512->1024, 4, stride=2, pad=1)` + BN + LeakyReLU | `[B, 1024, 4, 4]` |
| 6 | `Conv2d(1024->1, 4, stride=1, pad=0)` + **Sigmoid** + `Flatten` | `[B]` |

- **`Conv2d` со `stride=2`** — обучаемый даунсемплинг (замена pooling): каждый
  блок вдвое уменьшает разрешение.
- **`LeakyReLU(0.2)`** вместо `ReLU`: при отрицательном входе градиент не зануляется
  полностью (`f(x) = 0.2x` при `x < 0`) — важно, чтобы сигнал доходил до генератора.
- **BatchNorm** на всех блоках, кроме первого (на вход дискриминатора BN не ставят).
- Финальная свёртка 4x4 сжимает `4x4 -> 1x1`, **`Sigmoid`** переводит логит в
  вероятность `[0, 1]`, `Flatten` убирает лишние оси -> выход формы `[B]`.

**Инициализация весов** (`weights_init`, применяется через `model.apply(...)`):
свёртки — `N(0, 0.02)`, веса BatchNorm — `N(1, 0.02)`, bias — нули. Это
DCGAN-стандарт; без него GAN заводится менее стабильно.

## 4. Алгоритм обучения GAN

Лосс — `nn.BCELoss()` для обеих сетей. Два *раздельных* оптимизатора Adam с
`lr=0.0001`, `betas=(0.5, 0.999)` (пониженный `beta1 = 0.5` — рекомендация DCGAN
для стабильности). На каждом батче выполняются два независимых шага.

### Шаг 1. Обучение дискриминатора

Цель — научить `D` отличать реальное от фейкового.

1. **Реальные изображения -> метка 1** (со сглаживанием `0.9`):
   `loss_d_real = BCE(D(real), 0.9)`.
2. **Фейковые изображения -> метка 0**: сгенерировать `fake = G(z)`,
   `loss_d_fake = BCE(D(fake.detach()), 0.0)`.
   `.detach()` критичен: при шаге `D` градиент **не должен** течь в генератор —
   мы обновляем только дискриминатор.
3. Суммарный лосс `loss_d = loss_d_real + loss_d_fake`, `loss_d.backward()`,
   `opt_d.step()`.

### Шаг 2. Обучение генератора

Цель — заставить `D` принимать фейки за реальные.

1. Прогнать те же (не детачнутые) фейки через дискриминатор: `D(fake)`.
2. **Подставить генератору целевую метку 1** для фейков (хотя «по-честному» они 0):
   `loss_g = BCE(D(fake), 1.0)`. Это и есть non-saturating loss — генератор
   максимизирует `log D(G(z))`, что даёт сильный градиент даже когда `D` уверен.
3. `loss_g.backward()`, `opt_g.step()`. Так как вызывается только `opt_g.step()`,
   параметры дискриминатора не меняются, хотя градиент прошёл через него.

### Псевдокод одной итерации

```
для каждого батча real_images:
    # --- D ---
    opt_d.zero_grad()
    loss_d_real = BCE(D(real_images), 0.9)            # реальные -> 1
    fake = G(noise)
    loss_d_fake = BCE(D(fake.detach()), 0.0)          # фейки -> 0
    (loss_d_real + loss_d_fake).backward();  opt_d.step()

    # --- G ---
    opt_g.zero_grad()
    loss_g = BCE(D(fake), 1.0)                        # фейки -> 1 (обмануть D)
    loss_g.backward();  opt_g.step()
```

## 5. Оценка качества генерации

Смотреть на картинки глазами субъективно. Задание предлагает один количественный
и один качественный метод.

### 5.1. Leave-one-out 1-NN classifier accuracy

**Идея.** Берём `N` реальных и `N` сгенерированных изображений, помечаем реальные
меткой 1, фейковые — 0. По схеме **leave-one-out** обучаем
`KNeighborsClassifier(n_neighbors=1)` на всех объектах, кроме одного, и проверяем
предсказание класса на отложенном объекте; так перебираем все объекты и считаем
итоговую accuracy (`sklearn.model_selection.LeaveOneOut`).

**Что измеряет.** Способность 1-NN отличить фейк от реального по ближайшему
соседу:

- **accuracy ~ 0.5 — идеал.** Если фейки статистически неотличимы от реальных,
  то ближайший сосед любого объекта с равной вероятностью реальный или фейковый,
  и классификатор угадывает на уровне монетки.
- **accuracy -> 1.0 — плохо.** Чем легче 1-NN отделяет фейки, тем дальше выборки
  генератора лежат от реальных, тем хуже генератор.

Полный leave-one-out на всём датасете — это `O(N^2)`, поэтому в решении взята
подвыборка (по 600 реальных и фейковых), а изображения уменьшены до 32x32 и
развёрнуты в векторы `3072` признаков — это допустимое и явно документированное
упрощение.

### 5.2. t-SNE визуализация распределений

[**t-SNE**](https://scikit-learn.org/stable/modules/generated/sklearn.manifold.TSNE.html)
снижает размерность многомерных векторов до 2D, сохраняя локальную структуру
(близкие объекты остаются близкими). Берём те же flatten-векторы реальных и
сгенерированных изображений, проецируем в 2D и рисуем scatter, раскрашивая точки
по происхождению.

**Как читать график.** Чем сильнее **перемешаны** синие (реальные) и красные
(сгенерированные) точки — тем ближе распределение генератора к реальному. Два
**обособленных** облака означают, что фейки систематически отличаются от реальных
лиц. t-SNE — качественная иллюстрация к количественной метрике 5.1: сильное
перекрытие облаков соответствует accuracy, близкой к 0.5.

## 6. Итоговые результаты прогона

Обучение: **35 эпох** на **3143 изображениях** (49 батчей по 64), устройство — MPS
(Apple Silicon), время полного прогона ноутбука ~28,5 минут (включая скачивание
архива 4,28 ГБ и распаковку).

### Динамика лоссов

| Этап обучения | loss_D | loss_G | Комментарий |
|---|---|---|---|
| Эпоха 1 (старт) | 0.6333 | 8.6448 | начальное состояние |
| Эпохи 3-18 | ~0.33 | ~38-48 | `D` доминирует, `loss_G` высокий |
| Эпоха 19-20 | ~0.94-0.95 | ~7-19 | «выравнивание»: `D` ослаб, `loss_G` упал |
| Эпоха 35 (финал) | **0.6044** | **5.2999** | устойчивое равновесие |

**Интерпретация.** Лоссы не сходятся монотонно к нулю, а колеблются — это норма
для GAN (см. раздел 1). Видны две фазы: до эпохи ~18 дискриминатор доминирует
(`loss_D ~ 0.33`, `loss_G` высок и шумный), после эпохи 19 происходит характерное
«выравнивание» — дискриминатор ослабевает (`loss_D` растёт до ~0.6-0.9), а
`loss_G` падает с ~40 до ~5 и стабилизируется. **Mode collapse отсутствует**: ни
один лосс не ушёл в ноль с неограниченным ростом другого, оба остаются в разумном
диапазоне — обучение стабильно.

### Качество генерации

| Метрика | Значение | Идеал | Вывод |
|---|---|---|---|
| Leave-one-out 1-NN accuracy | **0.8942** | ~0.5 | генератор пока заметно отличим |
| t-SNE | два разделённых облака | сильное перекрытие | согласуется с accuracy 0.8942 |

**Интерпретация.** Accuracy `0.8942` заметно выше идеальных `0.5` — 1-NN ещё
довольно легко отличает фейки от реальных лиц. Это ожидаемый результат для
простого DCGAN, обученного небольшое число эпох на ~3000 лицах: задание оценивает
*корректность реализации* GAN и методов оценки, а не фотореализм. t-SNE
подтверждает картину: облака реальных и сгенерированных изображений заметно
разделены — распределения пока различаются. Сгенерированные лица уже узнаваемы
(виден овал, глаза, нос, рот, телесные тона), но детали размыты, встречаются
артефакты и асимметрия.

---

## 7. Уроки и выводы

1. **GAN — это не оптимизация одного лосса, а поиск равновесия двух сетей.**
   Падение лосса к нулю — *плохой* признак (один из игроков «победил»). Здоровое
   обучение — колебание обоих лоссов в разумном диапазоне.
2. **Диапазоны генератора и данных обязаны совпадать.** `Tanh` на выходе
   генератора => `Normalize` в `[-1, 1]` для реальных изображений. Иначе
   дискриминатор различал бы их тривиально.
3. **`.detach()` при шаге дискриминатора — обязателен.** Без него градиент шага
   `D` потечёт в генератор и испортит обучение. Генератор и дискриминатор
   обновляются *раздельными* оптимизаторами.
4. **Non-saturating loss.** Генератору подставляют целевую метку 1 для фейков
   (вместо минимизации `log(1 - D(G(z)))`) — это даёт сильный градиент даже когда
   дискриминатор уверен, и борется с vanishing gradients.
5. **DCGAN-рецепты работают.** `ConvTranspose2d`/`Conv2d`-stride вместо pooling,
   `BatchNorm`, `LeakyReLU` в `D` / `ReLU` в `G`, инициализация `N(0, 0.02)`,
   label smoothing `0.9` — этот набор позволяет завести GAN на лицах стабильно,
   без коллапса.
6. **Качество нужно мерить количественно.** Leave-one-out 1-NN accuracy даёт
   объективное число (идеал ~0.5), t-SNE — наглядную качественную картину.
   Высокая accuracy и разделённые облака t-SNE — согласованный сигнал, что
   генератору ещё есть куда расти.
7. **Простой DCGAN != фотореализм.** За 35 эпох на 3000 лицах получаются
   узнаваемые, но размытые лица. Для качества StyleGAN-уровня нужны прогрессивный
   рост разрешения, более глубокая архитектура и куда более длительное обучение.

## 8. Полезные функции с примерами

### Транспонированная свёртка — блок генератора

```python
import torch
import torch.nn as nn

# ConvTranspose2d со stride=2 удваивает пространственный размер.
block = nn.Sequential(
    nn.ConvTranspose2d(128, 64, kernel_size=4, stride=2, padding=1, bias=False),
    nn.BatchNorm2d(64),
    nn.ReLU(inplace=True),
)
x = torch.randn(8, 128, 16, 16)
print(block(x).shape)   # torch.Size([8, 64, 32, 32]) — размер удвоился
```

### Свёртка со stride — блок дискриминатора

```python
# Conv2d(stride=2) вдвое уменьшает размер — обучаемый даунсемплинг.
block = nn.Sequential(
    nn.Conv2d(64, 128, kernel_size=4, stride=2, padding=1, bias=False),
    nn.BatchNorm2d(128),
    nn.LeakyReLU(0.2, inplace=True),
)
x = torch.randn(8, 64, 64, 64)
print(block(x).shape)   # torch.Size([8, 128, 32, 32]) — размер уполовинен
```

### BCE-лосс для шага дискриминатора и шага генератора

```python
bce = nn.BCELoss()
batch = 16

# Шаг D: реальные -> 0.9 (label smoothing), фейки -> 0.0
real_preds = torch.rand(batch)          # выход D на реальных (после Sigmoid)
fake_preds = torch.rand(batch)          # выход D на фейках
loss_d = bce(real_preds, torch.full((batch,), 0.9)) \
       + bce(fake_preds, torch.zeros(batch))

# Шаг G: фейки -> 1.0 (генератор хочет обмануть дискриминатор)
gen_preds = torch.rand(batch)
loss_g = bce(gen_preds, torch.ones(batch))
```

### DCGAN-инициализация весов

```python
def weights_init(m):
    """Conv -> N(0, 0.02); BatchNorm -> weight N(1, 0.02), bias 0."""
    name = m.__class__.__name__
    if name.find('Conv') != -1:
        nn.init.normal_(m.weight.data, 0.0, 0.02)
    elif name.find('BatchNorm') != -1:
        nn.init.normal_(m.weight.data, 1.0, 0.02)
        nn.init.constant_(m.bias.data, 0.0)

generator.apply(weights_init)
discriminator.apply(weights_init)
```

### Денормализация для отображения

```python
def denorm(img):
    """[-1, 1] (выход Tanh) -> [0, 1] для imshow."""
    return (img.clamp(-1, 1) + 1) / 2
```

### Leave-one-out 1-NN accuracy

```python
import numpy as np
from sklearn.neighbors import KNeighborsClassifier
from sklearn.model_selection import LeaveOneOut
from sklearn.metrics import accuracy_score

# X — flatten-векторы изображений, y — метки (1 реальные / 0 фейковые)
loo = LeaveOneOut()
pred = np.zeros(len(y), dtype=int)
for tr, te in loo.split(X):
    knn = KNeighborsClassifier(n_neighbors=1).fit(X[tr], y[tr])
    pred[te] = knn.predict(X[te])
print('LOO 1-NN accuracy:', accuracy_score(y, pred))   # идеал ~0.5
```
