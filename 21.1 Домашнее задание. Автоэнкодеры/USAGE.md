# Прикладное применение: автоэнкодеры (AE, VAE, CVAE, Denoising AE, Image Retrieval)

## 1. Vanilla Autoencoder (AE)

### Где применяется

**Anomaly detection в IT-мониторинге и индустрии.** Самое массовое применение AE в production — детекция аномалий. Идея простая: обучаем AE на «нормальных» данных (логи серверов в штатном режиме, телеметрия исправного оборудования, легитимные транзакции). На аномалии (DDoS-атака, отказ турбины, фрод) reconstruction error взлетает, потому что модель таких паттернов не видела. Триггер: `MSE(x, recon(x)) > threshold`. В отличие от классификаторов, не требует размеченных «плохих» примеров — что критично, когда «плохое» редкое и заранее неизвестное. Примеры в индустрии: Datadog/New Relic используют AE-подобные модели для anomaly detection в метриках; в нефтегазовой отрасли (Газпром нефть, Лукойл) — для предсказания отказа насосов и компрессоров по виброметрии; в банковском фроде — на временных рядах транзакций клиента.

**Feature extraction / embeddings для downstream-задач.** Если данных мало, а размеченных — ещё меньше, AE даёт способ выучить полезные представления unsupervised на больших неразмеченных корпусах. Дальше латенты $z$ используются как фичи для классификатора, кластеризации, retrieval. Например, в дерматологии (продолжение задания 16.1): размеченных снимков невусов — тысячи, неразмеченных — миллионы. Pretrain AE на всех, потом logistic regression на латентах — даёт лучше, чем supervised на маленьком размеченном датасете.

**Сжатие данных для хранения и передачи.** Если у вас 10М снимков 45×45×3 (как в LFW в задании) — это 27 ГБ. Латенты по 256 floats — 10 ГБ, в 2.7 раза меньше. Для retrieval, где не нужно восстанавливать исходное изображение, а только сравнивать «похожесть», хранятся только латенты. На масштабе Pinterest / Instagram это экономит петабайты.

### Где НЕ стоит применять

- **Вместо PCA на линейных данных.** Если структура данных близка к линейной (табличные данные, временные ряды без сложной нелинейной геометрии), PCA даёт то же самое в разы быстрее и интерпретируемо. AE — overkill.
- **Для генерации новых данных.** AE-латент не структурирован — sampling из $\mathcal{N}(0, I)$ даёт мусор (как показано в Части 1.4 задания). Workaround с подгонкой mean/std работает плохо: латенты разных классов мультимодальны, гауссиана этого не учитывает. Для генерации нужен VAE/GAN/Diffusion.
- **Если нужна интерпретируемость каждой размерности.** PCA даёт компоненты с понятным смыслом (первая — направление максимальной дисперсии), ICA — статистически независимые компоненты. AE-латент — black box.
- **Когда есть много размеченных данных.** Supervised pretrain почти всегда превосходит unsupervised AE на той же downstream-задаче. AE имеет смысл только когда labels мало или дорого.

### Типичные ошибки

1. **Bottleneck слишком маленький.** `latent_dim=8` для лиц 45×45×3 — модель не сможет восстановить детали, реконструкции будут «средним» лицом. Подбор: начать с `latent_dim ≈ sqrt(input_dim)`, потом тюнить.
2. **Bottleneck слишком большой.** `latent_dim=4096` для MNIST — AE становится identity-функцией, ничего не сжимает, для downstream-задач бесполезно. Должно быть строго меньше входа.
3. **Забыли нормализовать вход.** Пиксели 0-255 на входе + Sigmoid на выходе — лосс взрывается, не сходится. Деление на 255 обязательно.
4. **MSE для цветных изображений с мелкими деталями.** MSE усредняет ошибку по пикселям, поэтому даёт «мыльные» реконструкции. Для production-качества используют perceptual loss (LPIPS, VGG features) или adversarial loss (как в VAE-GAN).
5. **Игнорирование batch normalization.** Без BN на глубоких энкодерах градиенты разбегаются. Стандарт — Conv2d → BN → ReLU.

## 2. Variational Autoencoder (VAE)

### Где применяется

**Drug discovery: молекулярные генеративные модели.** ChemVAE (Gómez-Bombarelli, 2018), Junction Tree VAE — стандарт для генерации новых молекул с заданными фармакологическими свойствами. Encoder переводит SMILES-строку или молекулярный граф в континуальный латент, decoder восстанавливает. Поскольку латент гауссов, можно делать gradient-based optimization: «двигать» точку в латенте в направлении желаемых свойств (растворимость, токсичность, аффинность к мишени), декодировать — получать новую кандидатную молекулу. Альтернатива (генерация случайных молекул и фильтрация) дороже на порядки. В индустрии используют Insitro, Recursion Pharmaceuticals, Atomwise — все на VAE-подобных архитектурах. Российский Биокад экспериментирует с подобными подходами для биоинженерии.

**Imputation пропусков в табличных данных и временных рядах.** Когда в датасете 10-30% пропущенных значений (типично для медицинских записей, логов сенсоров, опросов), VAE на табличных данных (TabVAE, VAEM) учится восстанавливать пропуски с учётом корреляций между признаками. Лучше k-NN imputation и mean-imputation, потому что моделирует совместное распределение, а не маргинальные. В клинических исследованиях так заполняют недостающие лабораторные показатели пациентов; в финансах — пропуски в макроэкономических рядах разных стран.

**Generative data augmentation для редких классов.** В imbalanced классификации (фрод, медицинские патологии, дефекты в QA) часто 0.1% позитивов и 99.9% негативов. VAE обучается на позитивах, генерирует синтетические позитивные примеры, классификатор обучается на сбалансированном датасете. Альтернатива SMOTE (линейная интерполяция) часто проигрывает: SMOTE работает в исходном пространстве и создаёт нереалистичные комбинации фичей; VAE — в латентном, где интерполяция семантически осмысленная.

**Continuous representation learning для interpolation и semantic editing.** Латент VAE гладкий: соседние точки соответствуют семантически близким объектам. Это даёт интерполяцию между состояниями («утренний кофе → вечерний латте» в food delivery, плавный переход стиля одежды в e-commerce), morph-видео (смешение двух лиц, плавное старение). StyleGAN2-ADA и StableDiffusion-VAE используют похожую идею в production.

**Прогнозирование редких событий и uplift modeling.** VAE на временных рядах (например, RNN-VAE, Transformer-VAE) моделирует условное распределение будущего по прошлому. Для редких событий (отказ оборудования, churn) даёт калиброванные вероятности, в отличие от deterministic регрессоров. Применяется в predictive maintenance (Siemens, GE), churn prediction (Telstra, Vodafone), retail forecasting.

### Где НЕ стоит применять

- **Если нужна высокая визуальная детализация.** VAE даёт размытые изображения — это известная проблема, следствие L2-подобного reconstruction loss (BCE/MSE на пикселях). Для фотореалистичных результатов берут GAN (StyleGAN3), Diffusion (Stable Diffusion, DALL·E, Imagen), VAE-GAN гибриды.
- **Если данных много И есть размеченные target.** Supervised approach всегда даст лучшие метрики при достаточном количестве labels. VAE применим, когда labels мало или вообще нет.
- **Discrete latents и категориальные данные.** Continuous gaussian latent плохо моделирует дискретные структуры (текст, графы, программный код). Берут VQ-VAE (discrete codebook), autoregressive модели или гибриды.
- **Сильно мультимодальные распределения.** Гауссова форма апостериорного $q_\phi(z|x)$ — упрощение. Если реальное распределение сильно мультимодально (например, кодируем сразу 1000 классов в маленький латент), VAE «усредняет» моды, даёт mode collapse. Решения — Normalizing Flows, IAF, hierarchical VAE.

### Типичные ошибки

1. **`reduction='mean'` в KL и BCE.** Самая частая ошибка. KL пропорционален `latent_dim`, BCE — числу пикселей. Если усреднять оба, KL «задушит» reconstruction (особенно при `latent_dim << pixels`), на выходе все реконструкции превратятся в средний пиксель. В коде задания используется `reduction='sum'` — обязательное условие.
2. **Неправильный знак в KL.** В литературе встречается две конвенции: $D_{KL}(q || p)$ и $D_{KL}(p || q)$, плюс перед минусом и минус перед минусом. В loss VAE формула с минусом: $D_{KL} = -\frac{1}{2}\sum(1 + \log\sigma^2 - \mu^2 - \sigma^2)$. Если знак инвертирован, лосс расходится за первые эпохи.
3. **Забыть про training/inference дихотомию в gaussian_sampler.** На training возвращаем $z = \mu + \sigma \cdot \varepsilon$ (стохастика), на inference только $\mu$ (mode). Если на inference тоже семплировать — реконструкции от запуска к запуску разные, retrieval становится нестабильным.
4. **Использовать logsigma vs sigma vs log_sigma_sq.** Конвенция сети: что именно она выдаёт — $\sigma$, $\log\sigma$, $\log\sigma^2$ или $\sigma^2$? Стандарт PyTorch — $\log\sigma^2$, тогда std = `exp(0.5 * logsigma)`. Путаница в туториалах идёт оттуда: одни проверяют через `torch.exp(logsigma)`, другие через `torch.exp(0.5 * logsigma)` — формулы KL у них разные.
5. **KL annealing забыт.** В первые эпохи лучше домножать KL на маленький коэффициент `beta` (или вообще на 0), постепенно увеличивая до 1. Иначе модель сразу стягивается к $\mathcal{N}(0, I)$ и не учится реконструкциям — «posterior collapse». Стандартный приём: linear annealing за 5-10 эпох.

## 3. Conditional VAE (CVAE)

### Где применяется

**Class-conditional generation в e-commerce и моде.** Сгенерировать товар определённого класса/стиля по запросу — «синее платье в стиле бохо», «кроссовки кэжуал размера М». Encoder получает изображение и метку (категория, цвет, размер), decoder может потом генерировать варианты при фиксированном style-латенте, меняя метку. Так делают Shopify Studio, Amazon AICreate, российский Lamoda Tech для генерации lookbook-ов. В отличие от обычного VAE, можно «направить» генерацию: получить именно красное платье, не белое.

**Inverse design в материаловедении и фарме.** Conditional VAE для молекул: condition — желаемые свойства (точка кипения, биодоступность, токсичность), decoder генерирует молекулу с этими свойствами. Это «обратная задача»: вместо «по молекуле предсказать свойства» — «по свойствам сгенерировать молекулу». BioMed.AI, российский ИВЦ им. Сеченова используют conditional flow models с похожей идеей для дизайна антител.

**Style transfer и controllable generation.** Один z кодирует стиль, condition кодирует контент (или наоборот). Применения — генерация одежды одного бренда в разных цветах (фиксируем style, меняем color-condition), генерация лиц одной этнической группы в разных эмоциях (style=identity, condition=emotion), генерация музыки одного композитора в разных жанрах. Магенту от Google и AIVA-композитора строят на похожих архитектурах.

**Few-shot generation с дополнительными атрибутами.** Когда классов много, но примеров каждого мало (long-tail), CVAE с условием по классу обобщает лучше: условие «направляет» декодер в нужную часть латента, нет необходимости заучивать каждый класс отдельно. Применяется в pose-conditional human generation (генерация человека в заданной позе по фото), text-to-image preview (Stable Diffusion в чем-то идейный преемник).

**Anomaly detection с учётом контекста.** Обычный AE детектирует аномалии по reconstruction error, но не учитывает контекст. Conditional AE: condition — категория объекта (например, время суток, тип сенсора, гео-зона). Реконструкция оценивается относительно «нормы для этой категории». В безопасности (Yandex Cloud Security, российский Kaspersky) применяется для contextual anomaly detection в логах: «логин из России в 3 ночи — норма для администратора, аномалия для бухгалтера».

### Где НЕ стоит применять

- **Если class labels noisy или сильно несбалансированы.** Если 50% меток ошибочны, condition даёт misleading signal — модель учит реконструировать «не то». Лучше сначала очистить labels (active learning, label denoising), потом обучать CVAE.
- **Если condition имеет high cardinality (10K+ значений).** One-hot encoding на 10K классов — 10K-мерный вектор на каждый пример. Лучше использовать embedding слой (как в NLP — обучаемое 64-мерное представление каждой категории).
- **Если condition continuous и нелинейный.** Для условий по числовым признакам (возраст, доход, время) one-hot не подходит. Используют MLP-проекцию или FiLM-conditioning (Feature-wise Linear Modulation).
- **Если нужна высокая визуальная детализация.** Те же проблемы, что у обычного VAE — размытые изображения. Для controllable high-quality generation сейчас стандарт — Conditional Diffusion (Stable Diffusion, ControlNet).

### Типичные ошибки

1. **Подавать condition только в encoder, не в decoder (или наоборот).** Модель не научится использовать condition в нужном месте: декодер не «знает» какой класс генерировать. В задании 21.1 condition подаётся в оба — `cat([x, y_oh])` на входе encoder и `cat([z, y_oh])` на входе decoder.
2. **One-hot для high-cardinality categorical.** Если у вас 1000 классов товаров, one-hot — 1000-мерный вектор, который доминирует над 32-мерным z. Решение — обучаемые embeddings (как `nn.Embedding(1000, 16)`), которые модель сама подгоняет под задачу.
3. **Не нормализовать condition.** Если condition — числовой признак с большим диапазоном (например, цена от 0 до 1000), его прямая конкатенация перетягивает градиент. Стандартизация обязательна.
4. **Слабая condition signal.** Если condition подаётся только в первый слой и теряется в глубине сети, модель его игнорирует. Решения — добавлять condition на каждом слое (concat или FiLM), использовать residual connections.
5. **Mode collapse по классу.** Модель генерирует все классы похоже, варьируя только мелкие детали. Признак: TSNE-кластеры по реальному class label и по predicted class сильно различаются. Решение — увеличить weight на condition-aware loss, проверить корректность форвард-прохода (точно ли decoder получает condition).

## 4. Denoising Autoencoder (DAE)

### Где применяется

**Image denoising в фотографии и медицинской визуализации.** Низкое освещение → высокий ISO → шум. Topaz Denoise AI, Adobe Lightroom Enhance, российский Nik Collection — все используют denoising AE/UNet под капотом. На медицинских снимках (low-dose CT для уменьшения радиационной нагрузки, низкокачественный MRI с короткой экспозицией, ультразвук с спекл-шумом) DAE восстанавливает диагностически значимые детали. Российская «Цифровая медицина», СберМедИИ применяют это для low-dose CT-скрининга лёгких.

**Audio denoising и voice enhancement.** Call centers (шумоподавление при разговоре оператора с клиентом), голосовые ассистенты (Алиса, Siri, Alexa — каждая встроенная в смартфон с микрофоном дальнего поля), music restoration (восстановление старых грампластинок). RNNoise (open-source), NVIDIA Maxine, Krisp — все на DAE-архитектурах на спектрограммах. На вход — спектрограмма с шумом, на выход — чистая.

**Self-supervised pretraining (denoising как задача).** Идея DAE — «дать модели грязный вход, заставить восстановить чистый» — фундаментальна для self-supervised learning. BERT (masked language modeling), MAE (Masked AutoEncoder для CV, He, 2021), data2vec, DINOv2 — все идейные потомки DAE. На неразмеченных данных модель учится reconstruct из corrupted версии, выученные представления переносятся в downstream-задачи через fine-tune. В индустрии так предобучают foundation models для CV, NLP, аудио — потом эти представления fine-tune на конкретные задачи с малыми размеченными выборками.

**Document restoration и удаление артефактов.** Восстановление старых документов (царапины, потёртости, выцветание), удаление JPEG-артефактов с web-изображений, удаление watermarks (этически спорно, но технически — типичная задача DAE). Российские ABBYY, FineReader используют похожие подходы для preprocessing документов перед OCR.

**Робастные представления для классификации.** Если данные на проде грязнее, чем на трейне (шум камеры, motion blur, occlusions), DAE-предобучение даёт энкодеру устойчивость. Модель «знает», что важная семантика остаётся одной и той же при добавлении шума. На MNIST + DAE pretrain валидационная точность лучше на 1-2%, на CIFAR — больше.

### Где НЕ стоит применять

- **Если шум структурированный, а не аддитивный гауссов.** DAE из коробки учит inverse process для добавленного gaussian noise. Для motion blur нужны специализированные deconvolution-сети (DeblurGAN, MPRNet), для compression artifacts — ARCNN, для rain/snow removal — Restormer.
- **Если ground truth clean data недоступен.** DAE требует пары (noisy, clean) для обучения. Если у вас только noisy data (типично в radio astronomy, low-light reality), используют Noise2Noise (Lehtinen, 2018), Noise2Self, Noise2Void — обучают без clean target, используя только пары noisy-noisy.
- **Если нужна real-time скорость на edge.** Полноценный UNet-DAE медленный. Для мобильного денойза (камеры iPhone, Pixel) используют гораздо более компактные модели — MobileDenoise, FastDVDnet.
- **Если шум не разрушает семантику.** Если задача downstream устойчива к шуму (например, классификация на high-level features), DAE pretrain не даёт прироста — лучше обучать supervised сразу.

### Типичные ошибки

1. **noise_factor слишком большой.** При `noise_factor=2.0` исходный сигнал теряется в шуме, сеть не может восстановить. Стандарт — `0.1-0.5` (как в задании 21.1, где `noise_factor=0.5`).
2. **Забыли clip(0, 1) после добавления шума.** $X + 0.5 \cdot \mathcal{N}(0, 1)$ выходит за пределы [0, 1]. Если decoder выдаёт Sigmoid (как принято для MNIST), а target — за пределами [0, 1], лосс не сойдётся. Стандарт — `torch.clamp(X_noisy, 0, 1)`.
3. **Использовать тот же noise на каждом батче.** Если шум фиксированный (например, генерируется один раз для всего датасета), сеть запоминает паттерн шума. Решение — генерировать новый шум на каждом forward pass: `X + noise_factor * torch.randn_like(X)` внутри training step.
4. **Тестировать DAE только на том же типе шума, что и в обучении.** Если обучили на gaussian noise, а на проде встречается speckle noise (медицинский УЗИ) — модель не работает. Решение — обучать на смеси типов шума или domain-specific augmentations.
5. **Использовать Sigmoid output, но MSE loss.** Sigmoid + MSE — нелинейная комбинация, градиент малый около 0/1. Лучше Sigmoid + BCE (как для VAE) или Linear + MSE. В задании 21.1 на DAE используется Sigmoid + MSE, что работает на MNIST из-за специфики данных (binary-like).

## 5. Image Retrieval через AE/VAE-латенты и KNN

### Где применяется

**Reverse image search (Pinterest, Google Images, Яндекс.Картинки).** Пользователь загружает фото, система ищет похожие. Производственный pipeline: backbone-сеть (ResNet50, ViT, CLIP) преобразует изображение в embedding (latent), индекс ANN (FAISS, ScaNN, Milvus, Qdrant) находит top-K ближайших соседей за миллисекунды на базе из миллиардов. AE/VAE — учебный случай этой идеи, в production используют CLIP-embeddings (мультимодальная семантическая близость, не только визуальная) или специализированные supervised metric learning модели (ArcFace для лиц, EfficientNet+ArcFace для товаров).

**E-commerce visual search (Wildberries, Lamoda, ASOS).** Пользователь делает фото товара, система ищет в каталоге похожие. Особенно полезно для одежды и аксессуаров, где словесное описание не передаёт нюансы. В Wildberries Visual Search backbone fine-tune на собственном каталоге товаров (1M+ позиций), embeddings 256-512 размерности, FAISS HNSW для индексирования. Latency — 50-200 мс на запрос.

**Face recognition pipeline.** Это базовая задача биометрии: на регистрации лицо кодируется в embedding (latent), хранится в базе. На авторизации лицо снова кодируется и сравнивается со всеми зарегистрированными по KNN. FaceNet, ArcFace, COSFace — все используют похожую схему. В России VisionLabs LUNA, NtechLab FindFace применяют это для skud-систем, банковской авторизации, поиска преступников. Critical: модель обучается не как AE (reconstruction), а как metric learning — что embeddings одного человека близки, разных — далеки.

**Duplicate detection и plagiarism check.** Поиск дубликатов в больших коллекциях: изображения с одинаковым контентом, но разным разрешением/кропом/JPEG-качеством. AE-латенты устойчивы к таким искажениям, потому что модель учится сжимать «семантику», а не пиксели. Применения — модерация UGC (TikTok, VK Клипы, ВКонтакте), DMCA-обработка (YouTube Content ID), академический plagiarism check для изображений (Turnitin, Антиплагиат).

**Content-based clustering без меток.** Если у вас 10K снимков продукции с конвейера и нужно понять «сколько у нас типов изделий», но меток нет — AE на пикселях + KMeans/DBSCAN на латентах даёт кластеры. Применяется в exploratory analysis для новых production lines, при анализе медицинских коллекций (типы клеток по морфологии), биодайверсити-исследованиях (типы насекомых на trap-камерах).

**Recommender systems по визуальному сходству.** В дополнение к коллаборативной фильтрации и контент-фильтрации, визуальные embeddings дают «похожие товары». Особенно для cold start (новые товары без user-item-interactions): для нового товара берём embedding, рекомендуем пользователю, кто уже купил визуально похожее. Wildberries, Ozon, AliExpress используют это для cross-sell и recommendation widgets.

### Где НЕ стоит применять

- **На очень больших коллекциях (>1M) без специализированных ANN-индексов.** Базовый `sklearn.NearestNeighbors(algorithm='ball_tree')` (как в задании 21.1) хорош для прототипирования, но плохо масштабируется. На миллионах — FAISS (Meta), ScaNN (Google), Annoy (Spotify), HNSW. На миллиардах — Milvus, Pinecone, Weaviate как managed-сервисы.
- **Если нужна семантическая, а не визуальная близость.** Запрос «диван» должен возвращать диваны разных цветов и стилей. AE-латент даст «похожее по пикселям» — белый диван и белую кровать сочтёт одинаковыми. Для семантического поиска используют CLIP-embeddings (мультимодальные text+image), supervised fine-tune на категориях, или metric learning с triplet loss.
- **На high-dimensional embeddings (>1000) без понижения размерности.** Curse of dimensionality: в 1000-мерном пространстве все точки «далеки» друг от друга, KNN теряет дискриминативность. Решения — PCA/UMAP для понижения до 64-256 dim перед индексированием, или специализированные структуры (LSH для высокой размерности).
- **Если данные сильно различаются по доменам.** Latent, обученный на лицах, не работает для retrieval по машинам. Модель — domain-specific. Иногда помогает CLIP (универсальный embedder), но для production обычно fine-tune на собственном домене даёт лучше.

### Типичные ошибки

1. **Использовать `LSHForest` из старых туториалов.** Этот класс **удалён из sklearn 0.21+** как nondeterministic и плохо поддерживаемый. Замена — `NearestNeighbors(algorithm='ball_tree')` (как в задании 21.1) или специализированные библиотеки (FAISS, Annoy).
2. **Использовать euclidean distance вместо cosine на необученных латентах.** На AE без metric learning loss euclidean часто хуже cosine: длина латента кодирует «уверенность» модели, не семантику. Стандарт — нормализовать латенты (`F.normalize(z, p=2, dim=1)`) перед индексированием, использовать cosine similarity.
3. **Забыть пересчитать индекс после переобучения модели.** Latent space меняется при каждом обучении (даже на тех же данных из-за SGD-стохастичности). Если в проде ANN-индекс остался от старой модели, а новая модель кодирует иначе — retrieval даёт мусор. Версионирование модели + индекса обязательно.
4. **Сравнивать «raw image distance» vs «latent distance» неправильно.** В задании 21.1 показано, что MSE между двумя картинками лиц в пикселях нерепрезентативен (один пиксель сдвига = высокий MSE), а latent distance отражает семантическую близость. Это не всегда так: если AE недообучен или bottleneck большой, latent распадается на «копию» пикселей. Sanity-check: t-SNE на латентах должен показывать осмысленные кластеры.
5. **Hard negatives mining не делается на этапе обучения encoder.** Если encoder — обычный AE (как в задании), retrieval работает только для «легких» случаев. Для production-качества энкодер обучают metric learning loss (triplet loss, NT-Xent, ArcFace), специально подбирая hard negatives — примеры, которые модель сейчас плохо различает.

## 6. Сравнение подходов: когда какой автоэнкодер выбрать

| Задача | Лучший метод | Почему |
|---|---|---|
| Сжать данные для retrieval | AE | Минимум кода, понятная latent, хорошо комбинируется с KNN |
| Сгенерировать новые семплы | VAE / Diffusion | KL делает latent рабочим для sampling из N(0,I) |
| Сгенерировать класс-конкретные семплы | CVAE / Conditional Diffusion | Condition даёт контроль над выходом |
| Убрать шум из реальных снимков | DAE / UNet / Restormer | Прямое сопоставление noisy→clean, end-to-end обучение |
| Найти похожие картинки | AE/VAE + KNN (учебно) / CLIP+FAISS (prod) | Латенты как embeddings |
| Anomaly detection на нормальных данных | AE | Reconstruction error как indicator аномалии |
| Высококачественные изображения | StyleGAN3 / Diffusion (Stable Diffusion, Imagen) | VAE даёт «мыльно» из-за L2-loss |
| Категориальные / дискретные данные | VQ-VAE / Autoregressive | Continuous VAE не подходит для дискретного |
| Молекулы и графы | ChemVAE / GraphVAE / JT-VAE | Специализированные encoders для графов |
| Imputation табличных пропусков | TabVAE / VAEM | Совместное распределение, не только маргинальные |
| Self-supervised pretraining | MAE / DINO / data2vec | Идейные потомки DAE, лучше работают на масштабе |
| Style transfer и interpolation | VAE / StyleGAN | Гладкий латент для семантического морфинга |
| Калиброванная uncertainty | VAE / Bayesian NN | Вероятностный подход, в отличие от deterministic |

## 7. Decision framework: когда автоэнкодеры — правильный выбор

| Ситуация | Подход |
|---|---|
| Нужна compression + downstream feature extraction | AE с conv encoder |
| Нужна generation + interpolation | VAE / Diffusion |
| Нужна generation с control по классу/атрибуту | CVAE / Conditional Diffusion |
| Нужно robust feature learning из noisy data | DAE / MAE |
| Нужен retrieval по визуальному сходству на учебной задаче | AE/VAE + KNN |
| Нужен retrieval в production на >1M | CLIP / metric learning + FAISS |
| Нужна anomaly detection на нормальных данных | AE с MSE как anomaly score |
| Нужна высокая визуальная детализация | GAN / Diffusion (не VAE) |
| Нужно понять конкретные признаки | PCA / ICA / Sparse AE / β-VAE |
| Нужна self-supervised foundation model | MAE, DINO, data2vec (потомки DAE) |
| Нужна generation discrete структур (текст, граф, код) | VQ-VAE, autoregressive, Diffusion |
| Нужна imputation табличных данных | TabVAE, VAEM |

## 8. Чек-лист практика

| Задача | Рекомендуемый подход |
|---|---|
| «Уменьшить размер 100K изображений до 256-D для быстрого поиска» | AE с conv encoder, `latent_dim=256`, MSE loss, сохранить веса |
| «Сгенерировать новые лица, похожие на наш датасет» | VAE с conv encoder, latent_dim=128, BCE+KL с `reduction='sum'`; для prod-качества — StyleGAN3 |
| «Создать варианты товара в разных цветах при фиксированном силуэте» | CVAE с цветом как one-hot condition, добавлять concat в encoder И decoder |
| «Очистить аудио в call-центре от фоновых шумов» | DAE на mel-спектрограммах, batchnorm + UNet-skip-connections, обучить на парах (noisy, clean) из синтетических смесей |
| «Найти 10 похожих картинок по запросу в каталоге 5K товаров» | AE+KNN с `NearestNeighbors(algorithm='ball_tree', metric='euclidean')` |
| «Найти 10 похожих картинок в каталоге 5M товаров» | CLIP-embeddings + FAISS HNSW, латенты нормализовать и cosine-similarity |
| «Найти подозрительные транзакции среди миллионов» | AE с tabular данными, MSE reconstruction error как anomaly score, threshold по 99-перцентилю на train |
| «Восстановить старые фото с царапинами» | DAE на парах (corrupted, clean), специализированный Restormer/SwinIR лучше для prod |
| «Сгенерировать молекулы с заданной растворимостью» | Conditional VAE на SMILES или molecular graph (ChemVAE, JT-VAE) с растворимостью как condition |
| «Понять структуру кастомерной базы без labels» | AE на табличных данных + KMeans/DBSCAN на латентах + t-SNE для визуализации |
| «Aligned face generation для маркетинга» | StyleGAN3 / Stable Diffusion с ControlNet (не VAE — слишком размыто) |
| «Уменьшить размер CT-снимков для архивирования» | AE с perceptual loss (LPIPS), не просто MSE — нужна сохранность диагностически важных деталей |
| «Денойз low-dose CT для уменьшения радиационной нагрузки» | UNet-DAE с парами (low-dose, full-dose), L1 loss + adversarial для деталей |
| «Pretraining foundation model на 100M неразмеченных изображений» | MAE (Masked AutoEncoder) или DINOv2, не классический DAE — на больших данных они лучше |
| «Imputation пропусков в медицинских записях» | TabVAE / VAEM, не просто mean imputation |
| «Anomaly detection в логах сервера» | AE на временных рядах (LSTM-AE или TCN-AE), порог по MSE на нормальной выборке |
| «Контекстная anomaly detection (учёт времени суток, пользователя)» | Conditional AE с context features как condition |
| «Visual search для пользовательской фотографии товара» | CLIP или fine-tuned ResNet50+ArcFace на каталоге + FAISS, AE недостаточно |
| «Style transfer одежды (тот же фасон в другом цвете)» | VAE с disentangled style/content (β-VAE) или CVAE с цветом как condition |
| «Калиброванная uncertainty в медицинской диагностике» | VAE с MC-dropout, не deterministic classifier — нужны вероятности |

## 9. Сравнение: AE vs PCA vs UMAP vs t-SNE для понижения размерности

| Метод | Когда выбирать | Плюсы | Минусы |
|---|---|---|---|
| **PCA** | Линейные данные, нужна интерпретируемость | Быстрый, детерминированный, понятные компоненты | Только линейные зависимости |
| **AE** | Нелинейные данные, нужны представления для downstream | Гибкий, мощный, инкрементальное обучение | Latent без структуры, нет sampling |
| **VAE** | Нелинейные + нужна generation/sampling | Структурированный латент, generation работает | Размытые реконструкции, сложнее обучать |
| **t-SNE** | Только визуализация в 2D/3D | Лучшие визуализации кластеров | Не для downstream, медленный, не инкрементальный |
| **UMAP** | Визуализация + быстрый embedding | Быстрее t-SNE, сохраняет глобальную структуру | Гиперпараметры чувствительны |
| **β-VAE** | Disentangled representations | Каждая ось — независимый фактор | Часто проигрывает по reconstruction |

**Практическое правило**: для exploratory analysis — UMAP/t-SNE; для production-embeddings под downstream — supervised pretraining поверх AE/CLIP; для generation — VAE/Diffusion; для интерпретации факторов — β-VAE или sparse AE.

## 10. Производственные паттерны и подводные камни

**Pattern: train-serve skew в нормализации данных.** AE обучается на нормализованных в [0,1] изображениях. На проде если на вход подают [0, 255], reconstruction error будет огромным — anomaly detection даст false positives. Сохраняйте препроцессинг pipeline вместе с моделью (TorchScript, ONNX), не только веса.

**Pattern: drift detection в anomaly detection.** AE-anomaly score уплывает с течением времени, потому что «нормальные» данные сами меняются (новые версии ПО → новые паттерны логов, новые продукты → новые типы транзакций). Решение — мониторинг распределения reconstruction error в проде, периодический retrain. Российский OZON делает это для fraud detection: ROC-AUC падает на 2-3% в месяц без retrain.

**Pattern: версионирование embeddings.** Если вы обновили AE-модель, все embeddings в индексе теперь несовместимы со старыми (модель видит мир иначе). Стратегии — двойной индекс (старый + новый, постепенный переход), полный rebuild (для маленьких баз), incremental encoding и проверка обратной совместимости (для очень больших — Pinterest, Spotify используют такое).

**Pattern: A/B testing генеративных моделей.** Невозможно объективно сравнить «качество» VAE vs Diffusion на одном датасете без user study. В продакшене (image generation для рекламы, фотофильтры) делают A/B с реальными пользователями — CTR на сгенерированных креативах, share rate, complaint rate. Pure technical metrics (FID, IS) часто не коррелируют с UX.

**Pattern: privacy при face retrieval.** В России и ЕС биометрия — регулируемая область (152-ФЗ, GDPR). AE на лицах нельзя обучать без согласия. Решения — federated learning, differential privacy в обучении, hashing embeddings перед хранением. Российский СберФейс использует homomorphic encryption для биометрической авторизации.

**Pattern: cold start в recommender systems.** Visual embeddings (AE/CLIP) решают cold start для новых товаров: нет user-item-interactions, но есть фото товара. Рекомендуем «похожих» по визуальному сходству до накопления реальных взаимодействий. Это hybrid recommender: visual KNN для cold, collaborative для warm.

**Pattern: data leakage в retrieval.** Если на trial-set у вас есть дубликаты trial-image и retrieval-base, KNN даёт «идеальный» результат, который не воспроизведётся на новых данных. Обязательная санити-проверка — дедуплицировать train/val/test по hash или perceptual hash перед обучением и оценкой.

---

**Итог.** Автоэнкодеры — это семейство методов, объединённое идеей «выучить компактное представление через reconstruction». В практическом применении: AE — для compression и anomaly detection, VAE — для generation и imputation, CVAE — для controllable generation, DAE — для денойза и self-supervised pretraining, retrieval через латенты + KNN — для visual search. В каждой области есть state-of-the-art альтернативы (Diffusion для generation, CLIP для retrieval, специализированные модели для денойза), но автоэнкодеры остаются важной базой — и архитектурно (encoder-decoder парадигма повсеместна), и образовательно (на них понимается всё остальное в генеративных моделях).
