# Конспект: Семантическая сегментация изображений — SegNet, UNet, лоссы для масок

## Введение

**Семантическая сегментация (semantic segmentation)** — задача попиксельной классификации изображения: каждому пикселю $x_{ij}$ ставится метка класса. На выходе получается карта той же ширины и высоты, что и вход, но с числом каналов, равным числу классов (или с одним каналом для бинарной задачи). В отличие от обычной классификации, здесь результатом является не одно число на картинку, а полноценная маска.

Сегментацию полезно отличать от трёх соседних задач:

- **Image Classification** — одна метка на всё изображение («это кошка»).
- **Object Detection** — для каждого объекта возвращается рамка (bbox) и класс. Размер рамки гораздо грубее реальной формы объекта.
- **Instance Segmentation** — попиксельная маска для каждого экземпляра отдельно (две кошки = две разные маски). Семантическая же сегментация склеивает все кошки в один общий «класс кошек».

**Применение в медицине** — одна из главных мотиваций развития методов: автоматическая сегментация опухолей на КТ/МРТ, поражений кожи на дерматоскопических снимках, клеток на гистологических препаратах. Качество сегментации напрямую влияет на оценку площади/объёма поражения, что важно для диагностики и планирования лечения.

## Датасет PH2

[PH2 (ADDI Project, University of Porto)](https://www.fc.up.pt/addi/ph2%20database.html) — коллекция дерматоскопических снимков для исследования меланом и родинок.

| Свойство | Значение |
|---|---|
| Количество изображений | 200 |
| Формат файлов | BMP (исходное разрешение разное) |
| Маски | бинарные: 0 = здоровая кожа, 1 = поражение |
| Целевой размер | 256x256 после `skimage.transform.resize` |
| Разбиение | train=100 / val=50 / test=50, фиксированное по `np.random.choice` с seed=42 |

**Подготовка данных** (по структуре `PH2 Dataset images/IMD_xxx/{IMDxxx_Dermoscopic_Image, IMDxxx_lesion}`):

1. Чтение через `skimage.io.imread`.
2. Resize 256x256 — для изображений с `anti_aliasing=True`, для масок с `anti_aliasing=False`, затем бинаризация по порогу 0.5.
3. Перенос каналов в начало: `np.rollaxis(X, 3, 1)` даёт `(N, 3, 256, 256)`. Маски расширяются `Y[..., np.newaxis]` до `(N, 1, 256, 256)`.
4. `DataLoader(list(zip(X, Y)), batch_size=25, shuffle=True)` — простейший вариант, не требующий собственного `Dataset`.

## Метрики качества сегментации

Задача попиксельная, поэтому accuracy — плохая метрика: при сильном дисбалансе (фон 90% — поражение 10%) тривиальный классификатор «всё фон» даёт accuracy = 0.9, ничего не сегментируя. Стандартом стали IoU и Dice.

### IoU (Intersection over Union, индекс Жаккара)

Считаем как отношение площади пересечения к площади объединения двух множеств — истинной и предсказанной маски:

$$\text{IoU} = \frac{|A \cap B|}{|A \cup B|} = \frac{TP}{TP + FP + FN}$$

где TP — пиксели, правильно отнесённые к классу `lesion`; FP — пиксели, ошибочно предсказанные как `lesion`; FN — пиксели поражения, которые модель пропустила. Идеальная сегментация даёт IoU = 1, полное промахивание — IoU = 0.

В PyTorch удобна `torchmetrics.JaccardIndex(threshold=0.5, task='binary')` — внутри сама применяет порог к вероятностям после сигмоиды и аккумулирует TP/FP/FN по батчам через `update`/`compute`. Между эпохами обязателен `iou_metric.reset()`, иначе статистика будет копиться от прошлой эпохи.

### Dice Coefficient (F1 для масок)

$$\text{Dice} = \frac{2|A \cap B|}{|A| + |B|} = \frac{2\,TP}{2\,TP + FP + FN}$$

Это попиксельная F1-мера. Через IoU выражается монотонной заменой:

$$\text{Dice} = \frac{2\,\text{IoU}}{1 + \text{IoU}}, \qquad \text{IoU} = \frac{\text{Dice}}{2 - \text{Dice}}$$

Обе метрики ранжируют модели одинаково (если Dice выше у одной модели, то и IoU выше). Различия:

- **Dice** даёт больший «бонус» за частичное совпадение (форма функции более пологая возле 1). Чаще используется в медицине, где модели надо «доброжелательно» сравнивать на сложных изображениях.
- **IoU** строже штрафует мелкие неточности — это «классическая» метрика VOC/COCO.

## Функции потерь для сегментации

### Binary Cross-Entropy (BCE)

Прямая формула BCE для одного пикселя:

$$\mathcal{L}_{BCE}(y, \hat y) = -\bigl[y \log \sigma(\hat y) + (1 - y)\log(1 - \sigma(\hat y))\bigr] \quad [1]$$

где $\hat y$ — **логит**, $\sigma(\hat y) = 1/(1+e^{-\hat y})$. Формула [1] численно нестабильна: при больших $|\hat y|$ значение $\sigma(\hat y)$ упирается в 0 или 1, и `log(0)` обращается в $-\infty$.

**Стабильная форма** получается подстановкой $\log\sigma(\hat y) = -\log(1+e^{-\hat y})$ и $\log(1-\sigma(\hat y)) = -\hat y - \log(1+e^{-\hat y})$:

$$\mathcal{L}_{BCE} = \hat y - y\hat y + \log(1 + e^{-\hat y}) \quad [2]$$

Дополнительно стабилизация делается через тождество

$$\log(1 + e^{-\hat y}) = \max(0, -\hat y) + \log(1 + e^{-|\hat y|})$$

— именно так считает `torch.nn.BCEWithLogitsLoss`. Поэтому в PyTorch правило: **всегда** использовать `BCEWithLogitsLoss`, а не `BCELoss(sigmoid(x), y)`. Они математически эквивалентны, но первая стабильнее при экстремальных логитах.

**Главная слабость BCE для сегментации** — она равноценно учитывает все пиксели. На медицинских снимках, где поражение занимает 10–20% площади, BCE ведёт к смещению в сторону класса фона: модель легко набирает низкий лосс, просто угадывая «фон везде».

### Dice Loss

Cделано из метрики Dice, чтобы напрямую оптимизировать качество маски:

$$\mathcal{L}_{Dice} = 1 - \frac{2\,|A \cap B| + \varepsilon}{|A| + |B| + \varepsilon}$$

Чтобы лосс был дифференцируем, в нём не делают threshold — вместо этого работают с **soft Dice**: $A = \sigma(\hat y)$ (вероятности), $B = y$ (one-hot маски). Малое $\varepsilon$ в числителе и знаменателе предотвращает деление на 0 и обеспечивает корректный градиент при пустых масках.

**Преимущества:**

- Не зависит от баланса классов — работает с относительной площадью пересечения.
- Прямо оптимизирует целевую метрику (или близкую к ней).

**Недостатки:**

- Шумные градиенты в начале обучения, когда маски почти пустые. Полезно стартовать с BCE и переключаться на Dice, либо комбинировать (см. Combo Loss).

### Focal Loss

Введена в [Focal Loss for Dense Object Detection (Lin et al., 2017)](https://arxiv.org/abs/1708.02002) для решения проблемы дисбаланса в детекции. Идея: пусть «лёгкие» примеры (которые модель уже предсказывает уверенно) дают меньший вклад в лосс, а «трудные» — больший.

$$\mathcal{L}_{FL}(y, \hat y) = -\alpha_t (1 - p_t)^\gamma \log(p_t), \qquad p_t = \begin{cases} \sigma(\hat y), & y = 1 \\ 1 - \sigma(\hat y), & y = 0\end{cases}$$

Параметр $\gamma$ (обычно 2) управляет «фокусом»: при $\gamma=0$ Focal Loss = BCE, при $\gamma>0$ модель уделяет всё больше внимания сложным примерам. $\alpha_t$ — балансировка классов, можно опустить ($\alpha_t = 1$).

В сегментации Focal Loss помогает, когда поражение очень мелкое, а большая часть пикселей фона уже хорошо классифицирована.

### Бонусные лоссы (краткий обзор)

| Лосс | Идея | Когда использовать |
|---|---|---|
| **Tversky Loss** [Salehi 2017](https://arxiv.org/abs/1706.05721) | Обобщение Dice: $\text{TP}/(\text{TP} + \alpha\,\text{FP} + \beta\,\text{FN})$. При $\alpha=\beta=0.5$ совпадает с Dice. Делая $\beta>\alpha$ (например 0.7/0.3), штрафуем FN сильнее FP — модель меньше пропускает поражение | Сильный дисбаланс, цена FN выше FP (медицина) |
| **Tversky-Focal** | Tversky -> $(1 - T)^\gamma$ — добавляем фокусирующий показатель | Очень редкие классы |
| **Combo Loss** | $\alpha \cdot \text{BCE} + (1-\alpha) \cdot \text{Dice}$ — линейная комбинация | Хочешь стабильности BCE и качества Dice одновременно |
| **Lovász-Hinge** | Суррогат для прямой оптимизации IoU через extension Lovász subdifferential | Двусторонняя сегментация, когда IoU — основная метрика |
| **Boundary Loss** [Kervadec 2018](https://arxiv.org/abs/1812.07032) | Расстояние от предсказания до границы истинной маски через `scipy.ndimage.distance_transform_edt` | Тонкие/нитевидные структуры, где важны границы |

В нашем решении реализованы все пять бонусных лоссов; для финального обучения был выбран **Tversky (alpha=0.3, beta=0.7)** — он показал стабильную сходимость без NaN и итоговый test IoU = 0.7691.

## Архитектуры для сегментации

### Encoder-Decoder парадигма

Любая современная сегментационная сеть использует двухуровневую структуру:

1. **Encoder** последовательно сжимает пространственное разрешение (через MaxPool или strided conv) и увеличивает число каналов. На выходе — низкоразмерное «сжатое» представление (bottleneck) с богатой семантикой, но потерянной геометрией.
2. **Decoder** восстанавливает пространственное разрешение через upsampling (transpose conv, MaxUnpool, bilinear) и постепенно уменьшает каналы.

Главный вопрос архитектуры — **как передать пространственную информацию из энкодера в декодер**, ведь bottleneck слишком «грубый». SegNet и UNet решают это по-разному.

### SegNet

**Архитектура** ([Badrinarayanan et al., 2015](https://arxiv.org/abs/1511.00561)) — encoder-decoder на базе VGG-16.

| Уровень | Encoder | Decoder |
|---|---|---|
| 0 | 3 -> 64 (depth=2) | 64 -> 1 (depth=2, last) |
| 1 | 64 -> 128 (depth=2) | 128 -> 64 (depth=2) |
| 2 | 128 -> 256 (depth=3) | 256 -> 128 (depth=3) |
| 3 | 256 -> 512 (depth=3) | 512 -> 256 (depth=3) |
| 4 (bottleneck) | 512 -> 512 (depth=3) | 512 -> 512 (depth=3) |

`depth` — количество свёрток 3x3 в блоке. После каждой свёртки идут BatchNorm + ReLU; в **последнем слое последнего декодера** их нет — это слой логитов.

**Ключевая идея SegNet — Pooling Indices.** Энкодер использует `MaxPool2d(return_indices=True)`, который возвращает не только уменьшенный feature map, но и индексы — позиции максимумов в исходной карте. Симметричный декодер применяет `MaxUnpool2d`, который раскладывает признаки строго обратно по этим запомненным позициям, оставляя нули в остальных. Это эквивалент «sparse upsampling»: мы не учим веса для апсемплинга, мы просто восстанавливаем геометрию, известную из энкодера.

```python
# В энкодере
self.maxpool = nn.MaxPool2d(kernel_size=2, stride=2, return_indices=True)
x, indices = self.maxpool(x)

# В декодере
self.maxunpool = nn.MaxUnpool2d(kernel_size=2, stride=2)
x = self.maxunpool(x, indices, output_size=size_before_pool)
```

`output_size` обязательно передавать — без него `MaxUnpool2d` может вернуть тензор не той формы (особенно для нечётных размеров).

**Сравнение с UNet:** SegNet передаёт между уровнями только индексы (это очень дёшево по памяти), а не сами активации. UNet же передаёт полные feature maps через skip-connections (более дорого, но точнее).

**Параметры**: ~29 млн (определяется главным образом четырьмя последовательно растущими уровнями свёрток).

**Преимущества:**

- Дёшев по памяти при инференсе — индексы занимают мало.
- Проще архитектура: симметричен, нет конкатенаций.

**Недостатки:**

- Информация из энкодера передаётся только через индексы — реальные значения признаков теряются. Декодер вынужден восстанавливать их «из ничего», что хуже работает на тонких границах.

### UNet

**Архитектура** ([Ronneberger et al., 2015](https://arxiv.org/abs/1505.04597)) — была изначально создана для биомедицинской сегментации, до сих пор является стандартом de facto в медицинских приложениях.

| Уровень | Encoder | Decoder |
|---|---|---|
| 0 | 3 -> 64 (DoubleConv) | concat[skip0, up] -> 128 -> 64 |
| 1 | 64 -> 128 (DoubleConv) | concat[skip1, up] -> 256 -> 128 |
| 2 | 128 -> 256 (DoubleConv) | concat[skip2, up] -> 512 -> 256 |
| 3 | 256 -> 512 (DoubleConv) | concat[skip3, up] -> 1024 -> 512 |
| 4 (bottleneck) | 512 -> 1024 (DoubleConv) | — |

**Ключевая идея UNet — skip-connections через конкатенацию.** Каждый уровень декодера получает не только апсемпленные признаки с нижнего уровня, но и **конкатенированные** фичи с симметричного уровня энкодера. Это даёт декодеру одновременно:

- богатую семантику с нижних слоёв (что это за объект),
- точную геометрию с верхних слоёв энкодера (где именно его границы).

```python
# В forward:
x = self.upconv(x)              # transpose conv: апсемплинг + halving channels
x = torch.cat([x, skip], dim=1) # конкатенация с feature map энкодера
x = self.double_conv(x)         # 2x (Conv 3x3 + ReLU)
```

**ConvTranspose2d vs Upsample+Conv для апсемплинга.** Два способа увеличить разрешение в 2 раза:

- `ConvTranspose2d(in, out, 2, stride=2)` — transposed свёртка с обучаемыми весами; иногда даёт «шахматные» артефакты.
- `nn.Upsample(scale_factor=2, mode='bilinear')` -> `Conv2d` — детерминированный билинейный апсемплинг + обычная свёртка; обычно мягче.

В нашем решении используется первый вариант (как в оригинальной статье).

**Параметры**: ~31 млн.

**Почему UNet стал стандартом для медицинской сегментации:**

1. Skip-connections сохраняют точные границы — критично для сегментации мелких структур (клеток, сосудов, поражений).
2. Хорошо работает при малом размере данных (медицинские датасеты редко превышают пару тысяч картинок).
3. Простая реализация: симметричная U-форма, понятные блоки.

## Выводы и уроки

1. **Универсальный baseline для бинарной медицинской сегментации**: UNet + BCEWithLogitsLoss + Adam(lr=1e-3) + CosineAnnealingLR. Дёшев, стабилен, не требует тонкой настройки.
2. **Когда BCE достаточно**: при умеренном дисбалансе (>10% foreground) BCE выигрывает у Dice/Focal по простоте и численной стабильности. Dice — резерв на случай, когда поражение занимает <5% площади.
3. **Skip-connections > Pooling indices**. Если выбирать между UNet и SegNet, UNet практически всегда лучше — за счёт лучшего сохранения геометрии.
4. **Гиперпараметры**: 40 эпох, batch=25, Adam lr=1e-3 с косинусным расписанием — стандартный набор. Для маленьких датасетов можно использовать аугментации (HorizontalFlip, RandomRotation, ColorJitter), но в нашем эксперименте они не использовались — даже без них UNet даёт IoU=0.85.
5. **Логиты vs вероятности**: все три кастомных лосса (BCE, Dice, Focal) принимают **логиты** и сами применяют сигмоиду внутри — стандарт PyTorch. Перед метриками IoU/Dice сигмоида применяется явно: `iou_metric(probs, yb.int())`.

## Полезные функции (с примерами кода)

### Универсальный train() с copy.deepcopy(state_dict)

```python
import copy

def train(model, opt, loss_fn, epochs, train_loader, valid_loader, device):
    best_iou = -1.0
    best_state = None
    for epoch in range(1, epochs + 1):
        model.train()
        for xb, yb in train_loader:
            xb, yb = xb.to(device).float(), yb.to(device).float()
            opt.zero_grad()
            logits = model(xb)
            loss_fn(logits, yb).backward()
            opt.step()

        # валидация: считаем val IoU
        model.train(mode=False)
        iou_metric.reset()
        with torch.inference_mode():
            for xb, yb in valid_loader:
                logits = model(xb.to(device).float())
                iou_metric.update(torch.sigmoid(logits), yb.to(device).int())
        val_iou = iou_metric.compute().item()

        if val_iou > best_iou:
            best_iou = val_iou
            best_state = copy.deepcopy(model.state_dict())

    model.load_state_dict(best_state)  # восстанавливаем лучшие веса
    return model, best_iou
```

### make_loss(name) — адаптер для разных сигнатур

Лоссы могут иметь разные порядки аргументов: `bce_loss(y_pred, y_real)`, `focal_loss(y_real, y_pred, gamma=2)`. Чтобы передавать их в `train()` единообразно как `loss_fn(logits, labels)`, оборачиваем:

```python
def make_loss(name):
    if name == 'BCE':
        return lambda logits, labels: bce_loss(logits, labels)
    elif name == 'Dice':
        return lambda logits, labels: dice_loss(logits, labels)
    elif name == 'Focal':
        # focal_loss принимает (y_real, y_pred) — меняем местами
        return lambda logits, labels: focal_loss(labels, logits, gamma=2)
    else:
        raise ValueError(name)
```

### distance_transform_edt для Boundary Loss

Boundary Loss требует «карту расстояний» от каждого пикселя до границы маски:

```python
from scipy.ndimage import distance_transform_edt

def make_distance_map(mask):
    # положительные значения внутри объекта, отрицательные — снаружи
    pos = distance_transform_edt(mask)
    neg = distance_transform_edt(1 - mask)
    return neg - pos  # signed distance map
```

Лосс умножает карту расстояний на (1 - softmax_pred) и суммирует — модель штрафуется за ошибки тем сильнее, чем дальше пиксель от истинной границы.

Большая часть инженерных усилий уходит не на выбор экзотического лосса, а на корректный пайплайн: численно стабильные функции потерь, правильный порядок применения сигмоиды, копирование лучших весов через `copy.deepcopy`, и аккуратные skip-connections / pooling indices в архитектуре.
