## Критерий 1. BCE Loss (0–3)
**Оценка: 3/3**

Реализованы обе версии бинарной кросс-энтропии: численно стабильная `bce_loss()` по формуле `y_pred - y_real * y_pred + log(1 + exp(-y_pred))` и «честная» `bce_true()` через сигмоиду с эпсилоном `1e-8`. Обе функции возвращают сумму по всем элементам и сопровождаются ассертами на тензорах формы `(3, 2)` и `(2, 1, 3, 3)`, а также на реальных логитах из `for_asserts/`. Результаты совпадают с эталонным `nn.BCEWithLogitsLoss(reduction='sum')` в пределах допуска. Критерий выполнен полностью.

## Критерий 2. SegNet (0–2)
**Оценка: 2/2**

Реализована полная encoder-decoder архитектура SegNet: 5 уровней `EncoderBlock` (3→64→128→256→512→512) c `MaxPool2d(return_indices=True)` и зеркальные 5 уровней `DecoderBlock` c `MaxUnpool2d`, использующим сохранённые индексы. Внутри блоков — последовательность `Conv2d → BatchNorm2d → ReLU` нужной глубины (2, 2, 3, 3, 3), финальный `decoder0` корректно выводит 1 канал логитов без BatchNorm/ReLU на последнем шаге. Логика «обрыва» последнего слоя через `if layer.out_channels == 1: break` несколько запутана — её можно было бы упростить отдельной веткой в `__init__`, — но она работает корректно.

## Критерий 5. Dice Loss (0–1)
**Оценка: 1/1**

`dice_loss()` реализован по корректной мягкой формуле: применяется `sigmoid` к логитам, считаются soft-`TP`/`FP`/`FN` без бинаризации, добавлен `eps=1e-8`, итоговая величина `1 - 2·TP/(2·TP + FP + FN + eps)`. Учтена и нормализация формы маски (`unsqueeze(1)` для случая `(N, H, W)`). Реализация эквивалентна стандартной из `segmentation_models_pytorch`.

## Критерий 6. Focal Loss (0–1)
**Оценка: 1/1**

`focal_loss()` написан строго по формуле `(1 - p_t)^γ · BCE` с `γ=2`: `p_t = σ(y_pred)·y_real + (1 - σ(y_pred))·(1 - y_real)`, BCE — численно стабильная версия. Возвращается сумма по элементам, что соответствует `torchvision.ops.sigmoid_focal_loss(..., alpha=-1, gamma=2, reduction='sum')`. Критерий выполнен полностью.

## Критерий 7. UNet (0–2)
**Оценка: 2/2**

Реализована классическая UNet с правильно устроенными блоками: `DoubleConv` (`Conv3×3 → BN → ReLU` дважды), `Down` (`MaxPool2d(2) → DoubleConv`), `Up` с `ConvTranspose2d(stride=2)` и конкатенацией skip-connection (с `F.pad` для выравнивания нечётных размеров) и `OutConv` (1×1 свёртка). Каналы 3→64→128→256→512→512 на энкодере и зеркально на декодере, skip-connection'ы передаются между симметричными уровнями. Архитектура корректна и работоспособна.

## Критерий 8. BONUS — лосс из статьи (0–5)
**Оценка: 2/5**

В коде определены три функции из бонусной части: `tversky_loss(α=0.7, β=0.3)`, `correlation_maximized_ssim_loss(...)` и `boundary_loss(...)` (через `scipy.ndimage.distance_transform_edt`). Формулы выглядят корректно, две из них — Tversky и Boundary — напрямую соответствуют статьям, упомянутым в задании. Однако ни одна из этих функций **не используется в обучении**: нет ни одного эксперимента, где модель тренировалась бы с бонусным лоссом, и, как следствие, нет ни Val IoU, ни Test IoU для сравнения с базовыми BCE/Dice/Focal. Главная цель бонуса — практически проверить альтернативный лосс и сопоставить результат с бейзлайном — не достигнута. За корректные реализации без экспериментальной части ставится 2/5; чтобы получить полный балл, достаточно прогнать `train()` хотя бы с одним из этих лоссов на SegNet или UNet и добавить строку в сравнительную таблицу.

## Общие комментарии

Базовая часть задания выполнена полностью и аккуратно: все шесть обязательных компонентов (BCE, SegNet, train, test, Dice, Focal, UNet) реализованы по эталонным формулам, ассерты против PyTorch-эталонов пройдены, обучены 6 комбинаций «модель × лосс», лучшие веса сохраняются. Сильные стороны — чистый структурированный код с docstring'ами, ассерты на численную близость к эталонным реализациям и наличие текстовых выводов после каждого раздела.

Слабые стороны:

1. **Только 20 эпох обучения** (вместо 40 в эталоне) — модели потенциально недо обучены. Это особенно заметно по Dice: SegNet+Dice=0.7533, UNet+Dice=0.7889 в Val IoU, тогда как при 40 эпохах Dice обычно догоняет BCE. Рекомендуется поднять `num_epochs` до 30–40.
2. **Test IoU посчитан только для SegNet+BCE (0.8035)**, для остальных пяти моделей в выводе только Best Val IoU. Для честного сравнения архитектур и лоссов нужно прогнать `test()` для всех 6 комбинаций и собрать сводную таблицу.
3. **Нет графиков** loss/IoU по эпохам, есть только текстовый лог. `matplotlib`-визуализация кривых обучения сильно облегчает анализ переобучения и сравнение лоссов.
4. **Нет содержательного анализа результатов**: на этой выборке UNet+BCE (0.8212) проиграл SegNet+BCE (0.8393) по Val IoU — нетипичное поведение, которое стоит обсудить (возможно, недо обучение UNet за 20 эпох, шум малой валидационной выборки в 50 изображений, отсутствие аугментаций). Так же стоит прокомментировать просадку Dice.
5. **Бонус**: реализации трёх лоссов корректны, но без обучения и метрик это лишь половина работы. Достаточно одного эксперимента с Tversky или Boundary, чтобы получить заметно больше баллов.

В целом работа аккуратная, базовая часть закрыта на максимум, основные потери — в недореализованной бонусной части и неполном тестировании моделей.
