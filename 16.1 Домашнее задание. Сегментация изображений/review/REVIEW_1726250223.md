# Рецензия на работу #1726250223

## Критерий 1. BCE Loss (0–3)
**Оценка: 3/3**

Реализованы обе версии бинарной кросс-энтропии: численно стабильная `bce_loss()` по формуле `y_pred - y_real * y_pred + log(1 + exp(-y_pred))` и «честная» `bce_true()` через `sigmoid` с эпсилоном `1e-8` для защиты от `log(0)`. Обе функции возвращают сумму по всем элементам (`reduction='sum'`) и сопровождаются полным набором ассертов: на синтетических тензорах формы `(3, 2)` и `(2, 1, 3, 3)`, а также на реальных логитах из `for_asserts/`. Сравнение проведено и с `nn.BCELoss(reduction='sum')`, и с `nn.BCEWithLogitsLoss(reduction='sum')` — все проверки сошлись в пределах допуска. Критерий выполнен полностью.

## Критерий 2. SegNet (0–2)
**Оценка: 2/2**

Реализована полная encoder-decoder архитектура: `EncoderBlock` и `DecoderBlock` с параметрами `(in, out, depth, kernel_size=3, padding=1)`, внутри блоков — последовательность `Conv2d → BatchNorm2d → ReLU` нужной глубины (2/2/3/3/3). 5 уровней энкодера с каналами 3→64→128→256→512→512 используют `MaxPool2d(2, return_indices=True)`, индексы корректно прокидываются в `MaxUnpool2d` зеркального декодера. Финальный `decoder0` создан с флагом `final_block=True` и выводит ровно 1 канал логитов без BatchNorm и ReLU, что соответствует требованиям задания. Архитектура чистая и работоспособная.

## Критерий 3. Код обучения (0–1)
**Оценка: 1/1**

Функция `train(model, train_loader, valid_loader, criterion, optimizer, device, num_epochs=10)` корректно реализует цикл обучения: переключение в train/eval-режимы, `optimizer.zero_grad → forward → loss.backward → optimizer.step`, накопление `running_loss`, вызов `test()` для валидации в конце каждой эпохи, печать `Train Loss / Val Loss / Val IoU` и индикатор прогресса через `tqdm`. Функция применена для обучения четырёх моделей (SegNet+BCE, SegNet+Dice, SegNet+Focal, UNet+BCE) по 30 эпох с Adam(lr=1e-4). Стоит отметить, что `history` метрик по эпохам не сохраняется и нет `best_val_iou`-чекпоинта (`copy.deepcopy(state_dict())`) — итоговая модель использует веса последней эпохи, а не лучшей; это не нарушает критерий формально, ранняя остановка по заданию не требуется, но в дальнейшем такой чекпоинт упростит честное сравнение моделей.

## Критерий 4. Код тестирования (0–1)
**Оценка: 1/1**

Функция `test(model, dataloader, criterion, device)` написана корректно: переводит модель в режим валидации, оборачивает проход в `torch.no_grad()`, считает `total_loss` и усредняет `JaccardIndex(threshold=0.5, task='binary', average='none')` по батчам, бинаризация порогом 0.5 встроена в саму метрику. Для SegNet+BCE и UNet+BCE функция вызвана на `test_dataloader` и даёт `Test IoU = 0.7822` и `Test IoU = 0.8478` соответственно. Однако для SegNet+Dice и SegNet+Focal `test()` фактически вызвана на `valid_dataloader`, а не на `test_dataloader`, поэтому Test IoU посчитан только для 2 из 4 обученных моделей — для полноты картины стоило прогнать все четыре эксперимента и собрать итоговую таблицу с тестовой метрикой.

## Критерий 5. Dice Loss (0–1)
**Оценка: 1/1**

`dice_loss(logits, labels, smooth=1e-8)` реализован по корректной мягкой формуле: применяется `sigmoid` к логитам, считаются soft-`TP = (preds * labels).sum()`, `FP = preds.sum() - TP`, `FN = labels.sum() - TP`, итоговая величина `1 - (2·TP + smooth)/(2·TP + FP + FN + smooth)`. Ассерт на численную близость к `smp.losses.DiceLoss(mode='binary')` пройден, лосс реально применён в обучении SegNet (Val IoU = 0.6369). Реализация эквивалентна стандартной из `segmentation_models_pytorch`.

## Критерий 6. Focal Loss (0–1)
**Оценка: 1/1**

`focal_loss(y_real, y_pred, gamma=2.0)` написан строго по формуле `(1 - p_t)^γ · BCE` с `γ=2`: `p_t = σ(y_pred)·y_real + (1 - σ(y_pred))·(1 - y_real)`, BCE — численно стабильная версия `y_pred - y_real*y_pred + log(1 + exp(-y_pred))`. Сигнатура `(y_real, y_pred)` обратная стандартной, и студент это явно учёл, оборачивая функцию в `criterion_focal = lambda logits, labels: focal_loss(labels, logits, gamma=2.0)` при подаче в `train()`. Ассерт против `torchvision.ops.sigmoid_focal_loss(..., alpha=-1)` пройден, лосс применён в обучении SegNet (Val IoU = 0.7176). Критерий выполнен.

## Критерий 7. UNet (0–2)
**Оценка: 2/2**

Реализована классическая UNet с правильно устроенными блоками: `DoubleConv` (`Conv3×3 → BN → ReLU` дважды), 4 уровня down (`MaxPool2d → DoubleConv`) с каналами 3→64→128→256→512→1024, 4 уровня up с `ConvTranspose2d(stride=2)`, конкатенацией skip-connection через `torch.cat` и DoubleConv, финальный `OutConv = nn.Conv2d(64, 1, 1)` для проекции в 1 канал. Модель обучена 30 эпох с `BCEWithLogitsLoss` и Adam(lr=1e-4); на отложенном тесте получено **Test IoU = 0.8478**, что практически совпадает с эталонным результатом (0.8481) и подтверждает корректность реализации. Лучший результат среди всех экспериментов студента.

## Критерий 8. BONUS — лосс из статьи (0–5)
**Оценка: 2/5**

Реализовано целых пять бонусных функций — это значительно больше требуемого минимума: `physiological_loss` (взвешенный BCE с морфологическими весами), `boundary_loss` через приближение distance transform лапласианом и гауссовым сглаживанием, `tversky_loss(α=0.7, β=0.3)`, `cmss_loss` (Correlation-Maximised SSIM с локальной корреляцией) и `topology_preserving_loss` по морфологическим скелетам, плюс корректные вспомогательные `ssim()`, `local_corr()`, `gaussian_kernel()`, `get_skeleton()`. Формулы выглядят аккуратно, две из них (Tversky, Boundary) напрямую соответствуют статьям из задания, а CMSS и topology-preserving — нетривиальные конструкции, которые сами по себе достойны уважения. Однако ни один из этих лоссов **не использован в обучении модели**: единственное упоминание — вычисление численных значений на dummy-сэмпле (BCE=5.97, Tversky=0.55, CMSS=0.85, TopologyPres=1.78 и т.д.), нет ни одного `train(..., criterion=tversky_loss, ...)`, нет ни Val IoU, ни Test IoU для бонуса, нет сравнения с базовыми BCE/Dice/Focal. Главная цель бонуса — практически проверить альтернативный лосс и сопоставить результат с бейзлайном — не достигнута. За пять корректных реализаций без экспериментальной части ставится 2/5; чтобы получить значительно больше баллов, достаточно прогнать `train()` хотя бы с одним из этих лоссов (проще всего с Tversky) на SegNet или UNet и добавить строку в сравнительную таблицу.

## Общие комментарии

Базовая часть задания выполнена полностью: все семь обязательных компонентов (BCE двумя способами, SegNet, train, test, Dice, Focal, UNet) реализованы по эталонным формулам, ассерты против PyTorch- и smp-эталонов пройдены, обучены 4 модели (SegNet с тремя лоссами + UNet с BCE). Особенно сильный результат — UNet+BCE с Test IoU = 0.8478 при 30 эпохах, что практически достигает эталонного значения 0.8481 при 40 эпохах: это говорит о корректной реализации архитектуры и грамотном выборе гиперпараметров. Структура ноутбука повторяет шаблон преподавателя, код читается легко.

Слабые стороны:

1. **Бонус не выполнен экспериментально** — реализованы пять (!) бонусных лоссов, но ни один не обучен. Это главная потеря баллов: достаточно одной строчки `train(SegNet(), train_loader, valid_loader, tversky_loss, ...)` и одного вызова `test()`, чтобы получить заметно больше за этот раздел.
2. **Test IoU посчитан только для 2 из 4 моделей.** Для SegNet+Dice и SegNet+Focal вызовы `test()` сделаны на `valid_dataloader`, а не на `test_dataloader`, поэтому фактического test-замера нет. Желательно прогнать `test()` для всех обученных моделей и собрать итоговую сводную таблицу.
3. **30 эпох вместо 40** — допустимо, но Dice и Focal явно недосходятся (Val IoU 0.64 и 0.72 против эталонных 0.79 и 0.81). Поднять до 40 эпох — минимальное усилие, которое заметно подтянет эти лоссы.
4. **Нет графиков loss/IoU по эпохам** ни для одной модели, только текстовый лог. `matplotlib`-визуализация кривых обучения сильно облегчает анализ переобучения и сравнение лоссов между собой.
5. **Нет финальной сводной таблицы и текстового анализа результатов.** Peer-review-проверяющий вынужден собирать цифры по тексту вручную; стоит добавить таблицу «модель × лосс → Val IoU / Test IoU» и 3–5 предложений с обсуждением (почему UNet > SegNet, почему Dice/Focal проседают, чего не хватает для дальнейшего роста).
6. **`train()` не сохраняет лучшие веса** — финальные метрики снимаются с весов последней эпохи. Добавить `best_val_iou` + `copy.deepcopy(model.state_dict())` и подгрузку лучшего чекпоинта в конце — стандартная практика, которая делает результаты более стабильными и воспроизводимыми.
7. **Глобальный random seed не фиксируется** (только `np.random.choice` для разбиения индексов, без `torch.manual_seed(42)`) — воспроизводимость экспериментов не гарантируется.

В целом работа аккуратная, базовая часть закрыта на максимум, реализация UNet особенно удачна. Основные потери — в недореализованной бонусной части (пять формул без единого обучения) и в неполном тестировании половины обученных моделей.
