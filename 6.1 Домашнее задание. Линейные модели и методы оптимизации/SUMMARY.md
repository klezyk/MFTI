# Конспект: Линейные модели и методы оптимизации

## Ключевые концепции

### Градиентный спуск
Итеративный метод оптимизации, при котором параметры обновляются в направлении антиградиента функции потерь: $w_{t+1} = w_t - \eta \cdot \nabla L(w_t)$. Скорость обучения $\eta$ (learning rate) контролирует размер шага — слишком большая приводит к расходимости, слишком маленькая — к медленной сходимости. На практике применяют стохастический (SGD) или мини-батчевый вариант, вычисляя градиент не по всей выборке, а по подмножеству (батчу).

### Логистическая регрессия
Линейная модель для задачи бинарной классификации, которая предсказывает вероятность принадлежности к классу 1 через сигмоидную функцию: $p = \sigma(w^T x) = \frac{1}{1 + e^{-w^T x}}$. Обучение сводится к минимизации функции кросс-энтропийных потерь (log-loss) градиентным спуском. Градиент имеет элегантную форму: $\nabla L = X^T (p - y)$, где $p$ — предсказанные вероятности, $y$ — истинные метки.

### Регуляризация Elastic Net
Комбинация L1 (Lasso) и L2 (Ridge) регуляризации, добавляемых к функции потерь: $L_{total} = L_{base} + \alpha \sum |w_j| + \beta \sum w_j^2$. L1-регуляризация обнуляет веса незначимых признаков (отбор фичей), L2 — штрафует большие веса и повышает устойчивость модели. Bias ($w_0$) никогда не регуляризуется, чтобы не смещать предсказания.

### Сигмоидная функция
Функция $\sigma(z) = \frac{1}{1 + e^{-z}}$, преобразующая любое вещественное число в интервал $(0, 1)$. Используется в логистической регрессии для получения вероятностей из линейной комбинации признаков. Её производная выражается через саму функцию: $\sigma'(z) = \sigma(z)(1 - \sigma(z))$.

### Стохастический градиентный спуск (SGD)
Вариант градиентного спуска, при котором градиент вычисляется не по всей выборке, а по случайному подмножеству (батчу). Это ускоряет обучение на больших датасетах и вносит шум, помогающий выходить из локальных минимумов. Размер батча — гиперпараметр, влияющий на баланс между скоростью и стабильностью обучения.

### Стандартизация признаков
Приведение признаков к нулевому среднему и единичной дисперсии: $x' = \frac{x - \mu}{\sigma}$. Критически важна для линейных моделей, так как градиентный спуск чувствителен к масштабу признаков — без стандартизации модель будет сходиться медленно или не сходиться вовсе. В sklearn реализована через `StandardScaler`.

## Модели

### Логистическая регрессия (реализация с нуля)

**Принцип работы:**
1. Линейная комбинация: $h = X \cdot w$
2. Сигмоида: $p = \sigma(h)$
3. Потери (cross-entropy): $L = -\sum [y_i \log p_i + (1-y_i) \log(1-p_i)]$
4. Градиент: $\nabla L = X^T(p - y)$
5. Обновление весов: $w \leftarrow w - \eta \cdot \nabla L$

**Плюсы:**
- Простая, интерпретируемая модель
- Выдаёт калиброванные вероятности
- Быстрое обучение и предсказание
- Хорошо работает на линейно разделимых данных

**Минусы:**
- Не может моделировать нелинейные зависимости
- Чувствительна к масштабу признаков
- Чувствительна к мультиколлинеарности (без регуляризации)

### Логистическая регрессия с Elastic Net

**Расширение функции потерь:**

$$L = -\sum [y_i \log p_i + (1-y_i) \log(1-p_i)] + \alpha \sum_{j=1}^{n} |w_j| + \beta \sum_{j=1}^{n} w_j^2$$

**Градиент регуляризации (bias $w_0$ не регуляризуется):**

$$\nabla L_{L1} = \alpha \cdot \text{sign}(w), \quad \nabla L_{L1}[0] = 0$$
$$\nabla L_{L2} = 2\beta \cdot w, \quad \nabla L_{L2}[0] = 0$$

**Плюсы (дополнительно к базовой):**
- L1 — отбор признаков (обнуление незначимых весов)
- L2 — устойчивость к мультиколлинеарности
- Комбинация даёт баланс между разреженностью и стабильностью

## Уроки и выводы

1. **Градиент — это сумма, не среднее.** В данной реализации градиент вычисляется как сумма по батчу ($X^T(p - y)$), без нормализации на размер батча. Это важно для прохождения тестов — формулу нужно использовать ту, что задана в задании.

2. **Bias не регуляризуется.** При добавлении L1/L2 регуляризации компоненты градиента для bias ($w_0$) обнуляются. Регуляризация bias сместила бы предсказания и ухудшила модель.

3. **Name mangling в Python.** Приватные методы с двойным подчёркиванием (`__loss`) преобразуются Python в `_ClassName__loss`. При наследовании для переопределения нужно использовать `_ParentClassName__loss`.

4. **StandardScaler критичен.** Без стандартизации пиксельных значений (0–255) градиентный спуск работает плохо — возникает overflow в exp. Pipeline (`make_pipeline`) гарантирует правильную последовательность преобразований.

5. **Неполный батч отбрасывается.** При генерации батчей последний неполный батч не возвращается — это стандартная практика для стабильности обучения.

6. **Sklearn-совместимость.** Наследование от `BaseEstimator` и `ClassifierMixin` позволяет использовать свой класс в `cross_val_score`, `Pipeline` и других инструментах sklearn.

def sigmoid(h):

def logit(x, w):
    return np.dot(x, w)

### Генератор батчей
```python
def generate_batches(X, y, batch_size):
    np.random.seed(42)
    perm = np.random.permutation(len(X))
    for start in range(0, len(X) - batch_size + 1, batch_size):
        idx = perm[start:start + batch_size]
        yield X[idx], y[idx]
```

### Pipeline с cross-validation
```python
from sklearn.pipeline import make_pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.model_selection import cross_val_score

pipe = make_pipeline(StandardScaler(), MyElasticLogisticRegression(0.01, 0.01))
scores = cross_val_score(pipe, X, y, scoring='accuracy', cv=5)
```

### Градиентный спуск 2D
```python
def grad_descent_2d(f, grad_f, lr, num_iter=100, r0=None):
    curr_r = r0.copy()
    history = []
    for _ in range(num_iter):
        history.append(np.hstack((curr_r, f(curr_r))))
        curr_r -= lr * grad_f(curr_r)
    return np.vstack(history)
```
