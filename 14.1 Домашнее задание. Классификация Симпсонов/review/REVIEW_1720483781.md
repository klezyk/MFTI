**Оценка: 15/15**

**Развернутое объяснение:**

Студент реализовал полный пайплайн классификации 42 персонажей мультсериала «Симпсоны»: загрузка данных, `SimpsonsDataset`, обучение двух моделей и формирование двух submission-файлов (`simple_cnn_baseline.csv` и `efficientnet_b0_submission.csv`). В качестве baseline используется предложенная в шаблоне задания `SimpleCnn` (5 свёрточных блоков, 180 762 параметра) — она достигает **val F1-micro = 0.7984** за 10 эпох с `Adam(lr=1e-3)`. В качестве финальной модели взята предобученная **EfficientNet-B0 (IMAGENET1K_V1)** с заменой последнего FC-слоя `classifier[-1]` на `Linear(1280, 42)` — полноценный transfer learning на все параметры.

Основная метрика финальной модели — **val F1-micro = 0.9773** (эпоха 10 из 10), что по шкале задания соответствует 15 баллам (порог ≥0.97). Мониторинг ведётся именно через `f1_score(..., average='micro')`, что полностью совпадает с метрикой Kaggle-конкурса — это важный плюс по сравнению с работами, где используется `weighted`. Оптимизатор — `Adam` с дефолтным lr, LR-scheduler — `ReduceLROnPlateau(patience=2)` по `val_loss`. Такой набор гиперпараметров для fine-tune pretrained-сети работает, хотя в эталонных рецептах обычно используют меньший lr (3e-4) и `AdamW(weight_decay=1e-4)` + `CosineAnnealingLR`.

Сильное инженерное решение — корректная борьба с дисбалансом классов через **WeightedRandomSampler** с весами `1 / classes_num[c]` и `replacement=True`, что компенсирует разрыв 3:2246 между редкими и частыми персонажами. Стратифицированный `train_test_split(test_size=0.25, stratify=...)` обеспечивает репрезентативную валидацию. Submission-файлы собраны корректно: 991 строка с колонками `Id` и `Expected`, классы декодированы через `label_encoder.inverse_transform`.

Что можно было бы улучшить: (1) **отсутствуют аугментации** — в `SimpsonsDataset.transform_images_to_tensors` train и val используют одинаковый пайплайн `(PILToTensor → ToDtype → Normalize → Resize)` без `RandomResizedCrop`/`HorizontalFlip`/`Rotation`/`ColorJitter`; это может давать потерю порядка +1-2% F1 и усиливать переобучение (train F1 = 0.9962 против val F1 = 0.9773 — заметный зазор); (2) **не применён TTA** — инференс это простой `argmax(logits)` без усреднения по оригиналу и горизонтальному флипу, что типично добавляет +0.5-1.0% F1; (3) **нет фиксации random seeds** — ни `torch.manual_seed`, ни `np.random.seed`, ни `random_state` в `train_test_split`, поэтому строгая воспроизводимость итогового Score не гарантирована (задание прямо требует воспроизводимости); (4) нет `classification_report` и confusion matrix, не проведён анализ самых путаемых персонажей; (5) графики train/val F1 построены, но без подробного комментария по динамике. Несмотря на перечисленные замечания, достигнутый val F1 = 0.9773 уверенно попадает в верхний интервал шкалы.

## Общие комментарии

**Сильные стороны:**
- Корректный transfer learning на EfficientNet-B0 с полной разморозкой и заменой головы — современный устойчивый baseline, val F1-micro = 0.9773.
- Правильная борьба с дисбалансом через `WeightedRandomSampler(weights = 1 / classes_num)` и стратифицированный split.
- Метрика мониторинга (`f1_score(..., average='micro')`) совпадает с метрикой Kaggle-конкурса; оба submission-файла (baseline и финальный) сформированы корректно.
- Чистая структура кода: переиспользуемые `train_epoch` / `val_epoch` / `train` / `predict`, отдельный класс `SimpsonsDataset`, визуализация предсказаний с вероятностями.
- Выполнены обе части задания: обязательный baseline SimpleCnn и улучшенная модель с transfer learning.

**Слабые стороны / что улучшить:**
- Отсутствуют аугментации на train (тот же transform, что и на val) — важнейший источник потери качества и главное расхождение с эталонной практикой для image classification.
- Не применён Test-Time Augmentation (усреднение softmax по `orig + hflip`) — простейший бесплатный +0.5-1% к F1.
- Нет фиксации random seeds и `random_state` в `train_test_split` — итоговый Score не строго воспроизводим, что прямо противоречит требованию задания.
- Для fine-tune предобученной сети обычно выбирают `AdamW(lr=3e-4, weight_decay=1e-4)` + `CosineAnnealingLR(T_max=epochs)` вместо `Adam(default)` + `ReduceLROnPlateau`.
- Не выведен `classification_report` и confusion matrix — для 42-классовой задачи с сильным дисбалансом анализ F1 по редким классам был бы полезен.
- В ноутбуке не приведён итоговый public F1 с Kaggle-лидерборда (ни числом, ни скриншотом), а имя команды на Kaggle формата `Имя_Фамилия_StepikID` в работе не упомянуто.
- Размер валидационной выборки `test_size=0.25` — разумный выбор, но `0.20` дал бы чуть больше данных для обучения.

**Рекомендации:**
- Разделить transform в `SimpsonsDataset` по режимам: для `train` добавить `v2.RandomResizedCrop([224,224], scale=(0.7,1.0))`, `v2.RandomHorizontalFlip(p=0.5)`, `v2.RandomRotation(10)`, `v2.ColorJitter(0.2, 0.2, 0.2, 0.02)`; для `val`/`test` оставить детерминированный `Resize + Normalize`.
- Реализовать простой TTA: `probs = 0.5 * softmax(model(x)) + 0.5 * softmax(model(hflip(x)))`, затем `argmax(probs)`.
- В начало ноутбука добавить блок фиксации сидов: `torch.manual_seed(42); torch.cuda.manual_seed_all(42); np.random.seed(42); random.seed(42)`, и `random_state=42` в `train_test_split`.
- Для финальной модели использовать `AdamW(pretrained.parameters(), lr=3e-4, weight_decay=1e-4)` и `CosineAnnealingLR(optimizer, T_max=epochs_num)`.
- Вывести `classification_report(y_val, y_pred, target_names=label_encoder.classes_)` и построить confusion matrix, чтобы увидеть, на каких персонажах модель ошибается.
- Добавить ячейку с печатью финального public F1 с Kaggle и именем команды `Имя_Фамилия_StepikID` для прозрачности peer review.
