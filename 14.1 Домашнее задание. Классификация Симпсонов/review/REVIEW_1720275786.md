**Оценка: 15/15**

**Развернутое объяснение:**

Студент (Kaggle-команда `Lev_Ogloblin_34317477` — формат `Имя_Фамилия_StepikID` соблюдён) реализовал полный пайплайн классификации 42 персонажей «Симпсонов»: загрузка архива с Google Drive, `SimpsonsDataset` в стиле ImageFolder, стратифицированный train/val split и обучение одной финальной модели. В качестве модели взята предобученная **ResNet50 с весами `IMAGENET1K_V2`** — современный вариант весов, дающий лучший baseline, чем `V1`. Голова заменена корректно: `model.fc = nn.Linear(2048, 42)`, при этом все 23.6M параметров сети размораживаются и учатся (полный fine-tune) — ровно та стратегия, которая устойчиво доводит качество до верхней границы шкалы.

Основной результат — **Kaggle Public F1-micro = 0.99574** (лучшая из двух сдач: 0.99149 → 0.99574), что значительно выше порога 0.97 и по шкале задания соответствует максимальным **15 баллам**. На валидации val accuracy стабилизируется на уровне **0.9734** (лучший 0.9734 на эпохах 9/10/12), val loss достигает минимума 0.1245 на 6-й эпохе и далее держится около 0.125. Разрыв train_acc = 0.9999 против val_acc = 0.9734 заметен — это классическое переобучение, но не мешает достичь высокого public F1, так как тестовая выборка Kaggle распределена похоже на общий пул обучающих изображений.

Гиперпараметры: `Adam(lr=1e-3)` + `StepLR(step_size=4, gamma=0.1)` (снижение LR в 10× каждые 4 эпохи), `CrossEntropyLoss`, 12 эпох, batch_size 128 (train) / 64 (val/test). Выбор `StepLR` с агрессивным шагом `gamma=0.1` фактически «замораживает» обучение после 5-й эпохи (видно по почти нулевому train_loss начиная с 6-й) — это одна из причин, по которой val accuracy перестаёт расти. В эталонных рецептах для fine-tune pretrained-сети обычно используют меньший `lr=3e-4` с `AdamW(weight_decay=1e-4)` и `CosineAnnealingLR(T_max=epochs)`, что даёт более плавную кривую и обычно +0.3-0.7% F1.

Препроцессинг минимален: `transforms.Compose([ToTensor(), Normalize(mean=ImageNet, std=ImageNet)])` + `image.resize((224,224))` через PIL. **Аугментации полностью отсутствуют** — модуль `v2` импортирован, но не задействован; ни `RandomResizedCrop`, ни `HorizontalFlip`, ни `ColorJitter`, ни `RandomRotation` не применяются. Также **не используется** `WeightedRandomSampler` или `class_weights` в loss — при реальном дисбалансе 3 vs 2246 изображений это упущенная возможность. То, что при столь базовой обвязке модель всё равно даёт public F1 ≈ 0.996, — заслуга веса `IMAGENET1K_V2` и полного fine-tune на 12 эпохах; с аугментациями и TTA результат был бы ещё стабильнее и, возможно, выше.

Воспроизводимость: `RANDOM_SEED = 42` зафиксирован только в `train_test_split(stratify=..., random_state=42)`. Ни `torch.manual_seed`, ни `np.random.seed`, ни `random.seed`, ни `torch.cuda.manual_seed_all` не заданы — строгая воспроизводимость public F1 не гарантирована (задание прямо требует воспроизводимости). Submission `resnet_12epochs.csv` собран корректно: колонки `Id` (`img0.jpg`, …) и `Expected` (имена классов после `inverse_transform`). TTA (усреднение softmax по `orig + hflip`) не применено. В ноутбуке нет `classification_report`, confusion matrix, графика train/val accuracy (есть только loss-кривая) и развёрнутого анализа результатов по персонажам.

## Общие комментарии

**Сильные стороны:**
- Отличный финальный результат: **Kaggle Public F1-micro = 0.99574** — значительно выше порога 0.97, соответствует 15/15 баллам.
- Корректный transfer learning на ResNet50 с современными весами `IMAGENET1K_V2` и полной разморозкой всех параметров.
- Стратифицированный `train_test_split(test_size=0.25, stratify=..., random_state=42)` — репрезентативная валидация.
- Грамотно подобранные batch_size (128/64) для Tesla T4 и правильная нормализация ImageNet-статистиками.
- Использование LR-scheduler (`StepLR`) для адаптивного снижения learning rate.
- Submission-файл собран в правильном формате `Id,Expected`; имя команды на Kaggle соответствует формату `Имя_Фамилия_StepikID`.
- Обе сдачи на Kaggle (0.99149 и 0.99574) демонстрируют осознанную работу над улучшением.

**Слабые стороны / что улучшить:**
- **Отсутствуют аугментации** на train — главный источник потенциального улучшения и главный фактор переобучения (train 0.9999 vs val 0.9734). Модуль `v2` импортирован, но не используется.
- **Не применён WeightedRandomSampler** и нет `class_weights` в CrossEntropyLoss — при дисбалансе 3:2246 это прямое упущение.
- **Нет TTA** (усреднение softmax по `orig + hflip`) — простейшее бесплатное улучшение +0.3-1% F1.
- **Не зафиксированы PyTorch/NumPy seeds** (`torch.manual_seed`, `np.random.seed`, `random.seed`, `torch.cuda.manual_seed_all`) — строгая воспроизводимость не гарантирована.
- Для fine-tune pretrained-сети обычно выбирают `AdamW(lr=3e-4, weight_decay=1e-4)` + `CosineAnnealingLR(T_max=epochs)` вместо `Adam(lr=1e-3)` + `StepLR(gamma=0.1)`; агрессивный StepLR фактически останавливает обучение после 5-й эпохи.
- Нет `classification_report`, confusion matrix и графика val accuracy — для 42-классовой задачи с сильным дисбалансом анализ по редким персонажам был бы полезен.
- Не реализована обязательная по шаблону задания модель `SimpleCnn` в качестве baseline — сразу обучается финальная ResNet50. Формально это лишает возможности сравнить baseline и финальную модель по той же метрике.
- Код слабо модуляризован (много inline-логики, минимальные комментарии), развёрнутого текстового анализа результатов и обоснования выбора гиперпараметров нет.

**Рекомендации:**
- Разделить transform по режимам: для train добавить `v2.RandomResizedCrop([224,224], scale=(0.7,1.0), antialias=True)`, `v2.RandomHorizontalFlip(p=0.5)`, `v2.RandomRotation(10)`, `v2.ColorJitter(0.2, 0.2, 0.2, 0.02)`; для val/test — детерминированный `Resize + Normalize`.
- Добавить `WeightedRandomSampler(weights = 1 / classes_num[label], num_samples=len(train), replacement=True)` в train `DataLoader` для компенсации дисбаланса.
- Реализовать TTA: `probs = 0.5 * softmax(model(x)) + 0.5 * softmax(model(hflip(x)))`, затем `argmax(probs)`.
- В начало ноутбука добавить блок фиксации сидов: `torch.manual_seed(42); torch.cuda.manual_seed_all(42); np.random.seed(42); random.seed(42); torch.backends.cudnn.deterministic = True`.
- Заменить оптимизатор и scheduler: `AdamW(model.parameters(), lr=3e-4, weight_decay=1e-4)` + `CosineAnnealingLR(optimizer, T_max=epochs)`.
- Добавить обязательный baseline `SimpleCnn` (из шаблона задания) с той же метрикой на val — это даст честное сравнение и подчеркнёт выигрыш transfer learning.
- Вывести `classification_report(y_val, y_pred, target_names=label_encoder.classes_)` и confusion matrix, чтобы увидеть, на каких персонажах модель ошибается (особенно на редких).
- Сохранять лучшие веса по val F1 через `copy.deepcopy(model.state_dict())`, а не веса последней эпохи — это страхует от деградации в конце обучения.
