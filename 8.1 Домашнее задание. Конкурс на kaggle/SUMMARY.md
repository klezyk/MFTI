# 8.1 Конкурс на Kaggle — Churn Prediction (Прогнозирование оттока клиентов)

## Ключевые концепции

### Бинарная классификация
Задача предсказания одного из двух классов (отток / не отток). В данном случае мы предсказываем не жёсткую метку (0 или 1), а **вероятность** принадлежности к классу 1 — это позволяет бизнесу самостоятельно выбирать порог принятия решений.

### ROC-AUC (Area Under the ROC Curve)
Метрика качества классификации, которая измеряет способность модели **ранжировать** объекты — то есть присваивать объектам класса 1 более высокие вероятности, чем объектам класса 0. ROC-AUC не зависит от порога классификации, что делает её удобной для соревнований и задач с несбалансированными классами. Значение 0.5 — случайное угадывание, 1.0 — идеальная модель.

### One-Hot Encoding
Способ преобразования категориальных признаков в числовые: для каждого уникального значения создаётся отдельная бинарная колонка (0 или 1). Параметр `drop_first=True` убирает одну из колонок (она линейно зависима от остальных), что важно для линейных моделей, чтобы избежать мультиколлинеарности.

### StandardScaler
Нормализация числовых признаков: вычитается среднее и делится на стандартное отклонение (`z = (x - mean) / std`). Линейные модели (LogisticRegression) чувствительны к масштабу признаков, поэтому нормализация критична. Для моделей на деревьях (CatBoost) нормализация не нужна.

### Кросс-валидация (Cross-Validation)
Метод оценки качества модели: данные делятся на K частей (фолдов), модель K раз обучается на K-1 частях и тестируется на оставшейся. Это даёт более надёжную оценку, чем однократное разбиение train/valid, так как каждый объект побывает в тестовой выборке ровно один раз.

### CatBoost
Библиотека градиентного бустинга от Яндекса, которая умеет работать с категориальными признаками **напрямую** (через параметр `cat_features`), без ручного кодирования. CatBoost использует ordered target encoding для категорий и symmetric trees, что обеспечивает высокое качество «из коробки».

### Регуляризация (параметр C в LogisticRegression)
Параметр C контролирует силу регуляризации: **чем меньше C, тем сильнее регуляризация** (штраф за большие веса). Регуляризация помогает бороться с переобучением. В scikit-learn C — это обратная сила регуляризации: `C = 1/λ`.

## Модели

### LogisticRegressionCV

**Принцип работы**: логистическая регрессия моделирует вероятность класса 1 через сигмоиду от линейной комбинации признаков. `LogisticRegressionCV` автоматически подбирает лучший параметр C через кросс-валидацию.

**Плюсы**:
- Быстрое обучение и предсказание
- Интерпретируемость (можно анализировать веса признаков)
- Хорошо работает с нормализованными числовыми данными

**Минусы**:
- Требует ручной предобработки (нормализация, one-hot encoding)
- Предполагает линейную разделимость
- Не может моделировать сложные нелинейные зависимости

**Результат**: ROC-AUC = **0.8451** (CV-5, лучший C=10.0)

### CatBoost

**Принцип работы**: ансамбль деревьев решений, обученных последовательно — каждое дерево исправляет ошибки предыдущих (градиентный бустинг). CatBoost использует ordered boosting и symmetric trees.

**Плюсы**:
- Работает с категориальными признаками «из коробки»
- Не требует нормализации
- Моделирует нелинейные зависимости и взаимодействия признаков
- Высокое качество по умолчанию

**Минусы**:
- Медленнее линейных моделей
- Менее интерпретируем
- Склонен к переобучению при большом количестве деревьев/высоком learning_rate

**Результат**: ROC-AUC = **0.8586** (valid, n_estimators=1000, learning_rate=0.01)

## Уроки и выводы

1. **Проверяй типы данных**: `TotalSpent` выглядел как числовой, но хранился как строка с пробелами. `data.info()` и `pd.to_numeric(errors='coerce')` помогают выявить такие проблемы.

2. **CatBoost — сильный baseline**: даже с параметрами по умолчанию CatBoost показал 0.8544, что лучше настроенной логистической регрессии (0.8451). Градиентный бустинг часто даёт «80% результата за 0% усилий».

3. **Learning rate и число деревьев взаимосвязаны**: маленький lr + много деревьев = медленное, но стабильное обучение. Большой lr + много деревьев = переобучение.

4. **ROC-AUC для несбалансированных классов**: метрика ROC-AUC устойчива к дисбалансу, в отличие от accuracy, который был бы ~74% даже при предсказании всех как «не отток».

5. **Predict_proba, не predict**: в соревнованиях по ROC-AUC нужно отправлять **вероятности** класса 1, а не жёсткие метки.

6. **Медиана из train для заполнения test**: при заполнении пропусков в тестовых данных используй статистики из train, чтобы не допустить утечки данных (data leakage).

```python

model = LogisticRegressionCV(

    refit=True,                            # Переобучить на всех данных с лучшим C
    cv=5,                                  # 5-fold CV
    random_state=42,
    max_iter=1000
)
model.fit(X_prepared, y)
```
