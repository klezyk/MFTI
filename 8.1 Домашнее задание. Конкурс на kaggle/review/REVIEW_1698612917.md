### Критерий 3 — Применение линейных моделей (3 балла)

**Оценка: 3/3**

LogisticRegression обучена через Pipeline с ColumnTransformer (StandardScaler для числовых, OneHotEncoder для категориальных), что является правильным подходом. Подбор гиперпараметра C выполнен через GridSearchCV по сетке [100, 10, 1, 0.1, 0.01, 0.001] с 5-fold StratifiedKFold и метрикой roc_auc. Лучший результат: C=1, ROC-AUC = 0.8413 (CV), 0.8545 (валидация) — результат сопоставим с эталонным решением (0.8451). Используется stratify при разбиении и random_state=42, что обеспечивает воспроизводимость. Все рекомендации критерия выполнены полностью.

## Общие комментарии

**Общее впечатление:** Работа демонстрирует уверенное владение инструментами машинного обучения. Студент корректно построил pipeline для линейной модели, грамотно использовал CatBoost с нативной обработкой категориальных признаков и получил результаты, сопоставимые с эталонным решением.

**Сильные стороны:**
- Грамотный pipeline для LogReg с ColumnTransformer (StandardScaler + OneHotEncoder)
- Корректный подбор гиперпараметров через GridSearchCV и StratifiedKFold
- Правильная обработка TotalSpent (конвертация типов, заполнение пропусков)
- Воспроизводимость: random_state=42 используется везде
- Код чистый, с комментариями на русском языке

**Главные недостатки:**
- Отсутствует анализ целевой переменной Churn и дисбаланса классов — это критически важный этап для задачи классификации
- Полностью отсутствуют текстовые выводы по EDA: графики построены, но не проинтерпретированы
- Модель CatBoost для финального submission обучена только на 80% данных (train split), а не на полном train — это снижает качество предсказаний

**Рекомендации по улучшению:**
- Добавить анализ целевой переменной: bar-plot или pie-chart с долями классов Churn, явно указать дисбаланс

- Перед генерацией submission переобучить лучшую модель на всём train для максимального качества
- Сохранять статистики (медиана, параметры скейлера) из train и применять к test, чтобы избежать data leakage
